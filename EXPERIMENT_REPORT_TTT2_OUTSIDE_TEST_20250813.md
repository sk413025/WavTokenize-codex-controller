# TTT2跨分支Outside音檔測試實驗報告
## 實驗編號: EXP_20250813_TTT2_OUTSIDE_TEST_LOAD_TRAINED_MODEL

### 📅 實驗時間
- **開始時間**: 2025年8月13日
- **實驗類型**: TTT2模型跨分支兼容性測試
- **實驗目標**: 驗證TTT2跨分支Outside音檔測試系統功能

### 🎯 實驗目標
1. **主要目標**: 實現TTT2模型的跨分支兼容性測試系統
2. **技術目標**: 
   - 自動檢測main分支和fix分支模型架構
   - 動態適配不同的ResidualBlock數量和正規化層類型
   - 智能模型載入和參數映射
   - 全面的音頻品質評估和報告生成

### 🛠️ 技術實現

#### 核心功能模組
1. **架構檢測系統** (`detect_model_architecture`)
   - 分析checkpoint的state_dict鍵名和形狀
   - 自動識別分支類型(main/fix)
   - 檢測ResidualBlock數量和維度配置
   - 識別正規化層類型(BatchNorm vs GroupNorm)

2. **動態模型創建** (`create_compatible_model`)
   - 根據檢測結果創建兼容的模型架構
   - 支援多種分支配置的模型patch
   - 智能正規化層選擇和參數配置

3. **智能模型載入** (`load_trained_model`)
   - 支援多種checkpoint格式(Lightning, best_model.pth)
   - 處理鍵不匹配和形狀不匹配問題
   - Fallback到基礎模型機制

4. **全面測試流程** (`test_model_on_audio`)
   - 音頻預處理和標準化
   - 模型推理和特徵提取
   - 多維度音頻品質評估

#### 分支架構差異
- **主分支(main)**: 5個ResidualBlocks, BatchNorm, 通常256→1024維度
- **修復分支(fix)**: 3個ResidualBlocks, GroupNorm, 通常512→1024維度

### 🧪 實驗設計

#### 實驗配置
- **測試模型**: TTT2基礎模型(未訓練狀態)
- **測試數據**: ./1n目錄下的outside音檔
- **測試檔案數**: 3個(初步) + 8個(完整)
- **音頻參數**: 24kHz採樣率, 48000樣本長度(2秒)
- **正規化設定**: auto(自動檢測) + groupnorm(強制)

#### 評估指標
1. **SNR (信噪比)**: 衡量增強後音檔的信號品質
2. **相關係數**: 原始和增強音檔的線性相關程度
3. **RMS差異**: 原始和增強音檔的均方根差異
4. **頻譜距離**: 頻域的歐幾里得距離
