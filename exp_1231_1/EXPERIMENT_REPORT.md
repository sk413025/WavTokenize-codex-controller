## 1. 實驗背景

### 1.1 特徵分析的核心發現 (exp_1231_feature)

| 發現 | 內容 |
|------|------|
| **噪音敏感度** | Mid-level (L5-L6) 最敏感 (sim=0.21-0.28)，Abstract (L13-L15) 魯棒 (sim=0.72-0.82) |
| **LoRA 變化分布** | 深層變化最大 (1.73)，淺層變化最小 (1.43) — **與預期相反** |
| **診斷結論** | 模型可能在深層「硬記」訓練集噪音模式，而非學習去噪 |

### 1.2 原始假設

```
理論預期 (降噪任務):
  - 淺層應該修改最多（因為噪音影響聲學特徵）
  - 深層應該變化最小（因為語義已經魯棒）

實際觀察 (LoRA 變化):
  - 淺層變化較小 (1.43-1.56)
  - 深層變化最大 (1.73) ❌

假設: 深層變化過大導致 Train/Val Gap
```

## 2. 實驗設計

### 2.1 Exp B: 凍結深層 (L9-L17)

**原理**: 如果深層變化過大是問題，凍結深層應該能消除 Train/Val Gap

```
配置:
  - LoRA 層: L0-L8 (9 層)
  - 凍結層: L9-L17 (9 層)
  - Rank: 256
  - 訓練: 300 epochs with curriculum learning
```

### 2.2 Exp E: 分階段訓練 (Progressive Training)

**原理**: 先讓淺層學會去噪，再逐步解凍深層

```
配置:
  - Phase 1 (Epoch 1-100):   只訓練 L0-L4 (5 層)
  - Phase 2 (Epoch 101-200): 訓練 L0-L8 (9 層)
  - Phase 3 (Epoch 201-300): 訓練 L0-L17 (18 層)
  - Rank: 256
```

## 4. 結果分析

### 4.1 Exp B 分析

**成功之處**:
- ✅ 凍結深層確實消除了 Train-Val Gap (0.06%)
- ✅ 驗證了「深層變化過大」與 Gap 有關

**失敗之處**:

- ❌ Acc 先升後降，典型的容量不足表現
- ❌ 模型無法真正學習降噪

**診斷**:
```
凍結深層後:
  - 淺/中層 LoRA 修正了特徵分布
  - 但深層期待「原始分布」的輸入
  - 分布不匹配 → 無法正確編碼

結果: Gap 消失（因為深層沒學東西）
      但 Acc 也很低（因為修正傳不過去）
```

## 6. 對特徵分析診斷的修正

### 6.1 原始診斷 vs 修正後理解

```
原始診斷 (exp_1231_feature):
─────────────────────────────
  觀察: 深層變化最大 (1.73)，淺層變化最小 (1.43)
  結論: 深層變化過大導致 overfitting
  建議: 凍結/限制深層

修正後的理解:
─────────────────────────────
  深層變化過大是「症狀」，不是「病因」

  真正的問題:
  - 淺層沒有充分學習去噪（變化太小）
  - 深層被迫「硬補償」淺層的不足
  - 這種補償是死記硬背訓練集，無法泛化

  正確的方向:
  - 不是「限制深層」，而是「強化淺層」
  - 深層需要「輕微調整」以適應淺層的修正
```

### 6.2 已驗證的假設

| 假設 | 驗證結果 | 說明 |
|------|----------|------|
| 深層變化過大導致 overfitting | ⚠️ 部分正確 | 凍結確實消除 Gap，但 Acc 極低 |
| 應該凍結深層只訓練淺層 | ❌ 錯誤 | 會導致分布不匹配 |
| 淺/深層可以分別學習 | ❌ 錯誤 | Phase 1-2 證明無效 |
| 深層與淺層需要協同訓練 | ✅ 正確 | Phase 3 才成功 |
