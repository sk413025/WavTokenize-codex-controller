# Exp Test: 淺層 LoRA 容量瓶頸測試報告

**日期**: 2026-01-15
**實驗目標**: 驗證 LoRA 容量是否是淺層學習不足的原因

---

## 一、背景與假說

### 1.1 問題觀察

基於先前實驗觀察到的矛盾：
- **降噪擾動在中層 (L5-L8) 最大** (cos_sim ≈ 0.21-0.29)
- **但 LoRA 訓練後深層變化最大** (深層 1.73 vs 淺層 1.43)

### 1.2 假說

**假說**: LoRA 容量不足導致淺層無法有效學習，深層被迫「硬補償」

### 1.3 驗證方法

- 只訓練 L0-L4 (5 層淺層)
- 凍結 L5-L17 (13 層)
- Loss 只監督 L4 輸出的 MSE
- 比較不同 LoRA rank (256/512/1024) 的 loss 下降程度

## 四、關鍵發現

### 4.1 結論：容量「不是」主要瓶頸

**重要發現**: 將 LoRA rank 從 256 增加到 512 (2 倍參數量)：
- Val Loss 僅改善 **0.25%** (51.79 → 51.66)
- CosSim 僅改善 **1.8%** (0.2101 → 0.2139)
- 兩者都在 **Epoch 48** 達到最佳，相同的收斂時間點

### 4.2 現象解讀

這個結果支持以下結論：

1. **容量不是主要瓶頸**
   - 2 倍參數量只帶來 ~0.25% 改善
   - 如果容量是瓶頸，預期會有顯著改善

2. **可能的真正原因**
   - 淺層特徵本身的表達能力限制
   - 任務難度的上限（淺層無法完全去噪）
   - 需要深層配合才能完成去噪任務

3. **對先前實驗的啟示**
   - 深層變化大不是因為淺層容量不足
   - 而是深層本身需要參與去噪過程
   - 中間層監督 (Exp K) 可能是更好的方向

### 4.3 與 Feature Map 分析的關聯

```
Feature Map 分析結論:
- L5-L8 (中層): 對噪音最敏感 (cos_sim ≈ 0.21-0.29)
- L13-L16 (深層): 對噪音最魯棒 (cos_sim ≈ 0.72-0.82)

本實驗發現:
- 只訓練淺層 (L0-L4) 的 CosSim ≈ 0.21
- 這與中層的噪音敏感度相當
- 說明淺層的去噪能力有其上限
```

## 五、後續建議

### 5.1 立即行動

1. **等待 r1024 結果**
   - 確認 4 倍參數量是否有更大改善
   - 如果仍無顯著改善，進一步確認容量非瓶頸

2. **關注 Exp K v2**
   - 中間層監督可能更有效
   - 直接指導噪音敏感層 (L4, L8)

### 5.2 實驗設計調整

1. **不需要繼續增加淺層容量**
   - r256 和 r512 結果幾乎相同
   - 增加容量不是解決方案

2. **應該關注的方向**
   - 中間層監督 (Exp K v2)
   - 全層訓練但差異化策略 (Exp I)
   - 深層參與的必要性

### 5.3 理論推論

```
結論: 去噪任務需要全層協同

淺層 (L0-L4): 識別噪音特徵
中層 (L5-L8): 噪音-語音分離
深層 (L9-L17): 語義保持與重建

只訓練淺層無法完成完整的去噪流程
```
