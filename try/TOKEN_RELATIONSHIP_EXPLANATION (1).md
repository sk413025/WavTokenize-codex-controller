## 🎯 核心概念總覽

```
原始音訊 → [Encoder] → 連續特徵 → [VQ Quantizer] → 離散 Token
                                          ↓
                                    Codebook Lookup
                                          ↓
                                    量化後的連續特徵
```

## 🔗 三者之間的因果關係

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    CAUSAL RELATIONSHIP                                   │
└─────────────────────────────────────────────────────────────────────────┘

1️⃣ 連續特徵 (Continuous Features)
   ↓
   ├─ 性質: Encoder 輸出的實數向量
   ├─ 形狀: (Batch, 512, Time)
   ├─ 值域: 實數 ℝ^512
   └─ 作用: 表示音訊的語義信息（未量化）

2️⃣ Codebook Embedding (可學習的查找表)
   ↓
   ├─ 性質: 預先學習的離散向量集合
   ├─ 形狀: (4096, 512)
   ├─ 值域: 實數 ℝ^512，但只有 4096 個選項
   └─ 作用: 提供離散化的"詞彙表"

3️⃣ 向量量化過程 (VQ)
   ↓
   連續特徵 ──[找最近鄰]──> Codebook ──[返回索引]──> 離散 Token
                                      └─[返回向量]──> 量化特徵

4️⃣ 離散 Token (Discrete Codes)
   ↓
   ├─ 性質: Codebook 的索引
   ├─ 形狀: (1, Batch, Time)
   ├─ 值域: 整數 {0, 1, 2, ..., 4095}
   └─ 作用: 壓縮表示，可傳輸、儲存、或用於 LLM 訓練

5️⃣ 量化後的連續特徵 (Quantized Features)
   ↓
   ├─ 性質: 從 Codebook 查表得到的向量
   ├─ 形狀: (Batch, 512, Time)
   ├─ 值域: 實數，但只能是 codebook 中的 4096 個向量之一
   └─ 作用: 用於解碼器重建音訊
```

## 📐 數學表示

### Vector Quantization 公式

```
給定：
- 連續特徵: z ∈ ℝ^D  (D=512)
- Codebook: C = {c₀, c₁, ..., c_{K-1}}  (K=4096, cᵢ ∈ ℝ^D)

量化過程:
1. 找最近鄰：
   k* = argmin_{k} ||z - cₖ||₂²
        k∈{0,...,K-1}

2. 輸出：
   - 離散 token: k* ∈ {0, 1, ..., 4095}
   - 量化特徵: q = c_{k*} ∈ ℝ^512

3. 損失函數：
   L_VQ = ||z - q||₂²  (reconstruction)
        + β||z - sg(q)||₂²  (commitment, sg=stop gradient)
```

### 完整編碼公式

```
輸入音訊: x ∈ ℝ^T  (T=24000 for 1 sec)
         ↓
Encoder: z = Encoder(x)  ∈ ℝ^{D×T'}  (D=512, T'=75)
         ↓
VQ:      對每個時間步 t ∈ {0, ..., T'-1}:
         - k_t = argmin_k ||z_t - c_k||²
         - q_t = c_{k_t}
         ↓
輸出:    
- Tokens: K = [k₀, k₁, ..., k_{T'-1}]  ∈ {0,...,4095}^{T'}
- Quantized: Q = [q₀, q₁, ..., q_{T'-1}]  ∈ ℝ^{D×T'}
```

## 💡 關鍵理解

### 1. **連續特徵 → 離散 Token**
```
連續特徵 (Continuous)           離散 Token (Discrete)
[0.23, -0.45, 0.89, ...]  →→→  2347
[0.15, 0.73, -0.22, ...]  →→→  891
[0.19, 0.92, -0.48, ...]  →→→  3102

512 個實數                       1 個整數
(高維空間中的一個點)            (codebook 的索引)
```

### 2. **Codebook 的角色**
```
Codebook 是一個「字典」：

    索引 (Token)  →  向量 (512-D)
    ──────────────────────────────
    0            →  [0.21, 0.88, -0.51, ...]
    1            →  [0.15, 0.73, -0.22, ...]
    2            →  [0.33, -0.12, 0.67, ...]
    ...
    2347         →  [0.23, -0.45, 0.89, ...]  ← 訓練學到的最佳表示
    ...
    4095         →  [-0.45, 0.23, 0.11, ...]

類比：
- Token = 字典中的頁碼
- Codebook vector = 該頁的內容
- VQ = 找到最接近的頁
```

### 3. **為什麼需要量化？**

```
優點：
✅ 壓縮: 512 floats (2KB) → 1 int (8 bytes) = 256× 壓縮
✅ 離散: 可用於語言模型 (LLM 需要離散 token)
✅ 穩定: 減少噪聲和過擬合
✅ 結構: 學習到的 codebook 捕捉音訊的典型模式

缺點：
❌ 信息損失: 512-D 空間 → 4096 個點 (離散化誤差)
❌ VQ Loss: 需要額外的訓練損失來優化 codebook
```

## 🧪 實驗驗證代碼

```python
import torch
from decoder.pretrained import WavTokenizer

# 載入模型
config_path = "./config/wavtokenizer_mediumdata_frame75_3s_nq1_code4096_dim512_kmeans200_attn.yaml"
model = WavTokenizer.from_hparams0802(config_path)
model.eval()

# 創建測試音訊
audio = torch.randn(1, 24000)  # 1秒音訊

# 1. 獲取連續特徵（通過 encoder）
with torch.no_grad():
    encoder_output = model.feature_extractor.encodec.encoder(audio.unsqueeze(1))
    print(f"連續特徵形狀: {encoder_output.shape}")  # (1, 512, 75)
    print(f"連續特徵範例 (t=0): {encoder_output[0, :5, 0]}")  # 前5維

# 2. 向量量化
with torch.no_grad():
    features, tokens = model.encode_infer(audio, bandwidth_id=torch.tensor([0]))
    print(f"\n量化後特徵形狀: {features.shape}")  # (1, 512, 75)
    print(f"離散 Token 形狀: {tokens.shape}")  # (1, 1, 75)
    print(f"Token 範例: {tokens[0, 0, :10]}")  # 前10個 token

# 3. 檢查 Codebook
quantizer = model.feature_extractor.encodec.quantizer
codebook = quantizer.vq.layers[0].codebook
print(f"\nCodebook 形狀: {codebook.shape}")  # (4096, 512)
print(f"Codebook[0] 範例: {codebook[0, :5]}")  # 第0個 codebook entry 的前5維

# 4. 驗證 Token → Codebook Lookup
token_idx = tokens[0, 0, 0].item()  # 第一個 token
codebook_vector = codebook[token_idx]  # 從 codebook 查找
quantized_vector = features[0, :, 0]  # 量化後的特徵

print(f"\n驗證 Token={token_idx} 的 Lookup:")
print(f"Codebook[{token_idx}][:5] = {codebook_vector[:5]}")
print(f"量化特徵[:5] = {quantized_vector[:5]}")
print(f"是否相等? {torch.allclose(codebook_vector, quantized_vector, atol=1e-5)}")

# 5. 計算量化誤差
with torch.no_grad():
    # 原始連續特徵 vs 量化特徵
    original = encoder_output
    quantized = features
    quantization_error = torch.mean((original - quantized) ** 2)
    print(f"\n量化誤差 (MSE): {quantization_error.item():.6f}")
```

## 📊 總結表格

| 項目 | 連續特徵 | Codebook | 離散 Token | 量化特徵 |
|------|----------|----------|------------|----------|
| **形狀** | (1, 512, 75) | (4096, 512) | (1, 1, 75) | (1, 512, 75) |
| **數據類型** | float32 | float32 | int64 | float32 |
| **值域** | 任意實數 | 任意實數 | {0,...,4095} | codebook 中的向量 |
| **來源** | Encoder 輸出 | 訓練學習 | VQ 輸出索引 | Codebook lookup |
| **用途** | VQ 輸入 | 查找表 | 壓縮/傳輸/LLM | Decoder 輸入 |
| **可學習** | ✗ (推理時) | ✓ (訓練時) | ✗ (量化結果) | ✗ (查表結果) |
| **信息量** | 高 (連續) | 中 (離散集合) | 低 (索引) | 中 (離散) |

---

## 🎯 一句話總結

**連續特徵**通過**向量量化 (VQ)** 找到 **Codebook** 中最接近的向量，該向量的**索引**就是**離散 Token**，而該**向量本身**就是**量化後的特徵**，用於解碼重建音訊。
