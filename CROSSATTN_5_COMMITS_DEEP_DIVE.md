## 二、基础知识：什么是「退化」？

### 2.1 注意力退化的定义

```
正常的注意力机制：
───────────────────────────────────────
每个 token 对不同的 speaker tokens 有不同的关注度

Token 1: [0.10, 0.40, 0.35, 0.15]  ← 各不相同
Token 2: [0.30, 0.15, 0.20, 0.35]  ← 有变化
Token 3: [0.25, 0.25, 0.28, 0.22]  ← 动态调整
...


退化的注意力机制 (S=1)：
───────────────────────────────────────
只有1个 speaker token → softmax 永远输出 1.0

Token 1: [1.0]  ← 完全一样
Token 2: [1.0]  ← 没有选择
Token 3: [1.0]  ← 完全固定
...

所有 token 得到完全相同的 speaker 信息！
→ 失去了 attention 的「动态选择」能力
```

### 2.2 如何度量退化？

```
关键指标：attn_w_std (注意力权重标准差)

attn_w_std = √[E[(w - w̄)²]]

其中 w 是所有 tokens 对所有 speaker tokens 的注意力权重


物理意义：
─────────────────────────────────────
• attn_w_std = 0    → 所有权重完全相同 → 退化 ❌
• attn_w_std > 0    → 权重有变化 → 正常 ✅

类比：
  标准差 = 0  就像所有学生都考相同分数（没有区分度）
  标准差 > 0  就像学生分数有高有低（有区分度）
```

### 3.2 数学解释：为什么 mean = 0.25？

```
K=4 的注意力机制：

每个 token 对 4 个 speaker tokens 做 softmax
softmax([s₁, s₂, s₃, s₄]) = [w₁, w₂, w₃, w₄]

性质：w₁ + w₂ + w₃ + w₄ = 1  (概率和为1)

如果 4 个 speaker tokens 的重要性大致相当，
则期望值：E[wᵢ] ≈ 1/4 = 0.25 ✅

类比：
  4 个学生平分一个披萨
  每人期望得到 1/4 = 25%

但实际分配会有变化：
  可能是 [15%, 40%, 30%, 15%]
  也可能是 [30%, 15%, 20%, 35%]
  等等...

平均值是 25%，但每次不同（std > 0）
```

# Commit 3: 现象分析 - Influence Breakdown

**Commit:** `e4872a6`
**标题:** [EXP-20251105] CrossAttn K=4 Influence Breakdown（C→W/W→C）

> 💡 这是最关键的突破！发现了"正负抵消"现象

---

## 一、实验动机

```
🎯 核心困惑：

   已知：K=4 解除了退化，注意力正常
   观察：Speaker 改变 → 63% tokens 翻转
   问题：但准确率只变化 ±0.3%

   ❓ 为什么大量翻转却没有带来准确率的显著变化？

   假设 H1: 翻转中有很多是"错→错"的无效翻转
   假设 H3: 可能有"对→错"和"错→对"互相抵消
```

---

## 二、基础知识：四象限分解理论

### 2.1 核心概念

对于每个 token 位置，有两次预测：
```
yᵢ        真实标签 (Ground Truth)
ŷᴺᵢ      Normal Speaker 的预测
ŷⱽᵢ      Variant Speaker 的预测 (Zero/Random)
```

根据两次预测的正确性，分为4类：

```
┌─────────────────────────────────────────────────────────────┐
│                   四象限分解矩阵                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│              Normal 预测正确    Normal 预测错误             │
│              ─────────────────  ─────────────────           │
│  Variant     │                 │                 │          │
│  预测正确    │   C → C         │   W → C         │          │
│              │  (都对)         │  (修正)         │          │
│              │  👍 保持好      │  👍 变好了      │          │
│              ─────────────────  ─────────────────           │
│  Variant     │                 │                 │          │
│  预测错误    │   C → W         │   W → W         │          │
│              │  (破坏)         │  (都错)         │          │
│              │  👎 变差了      │  😐 保持差      │          │
│              ─────────────────  ─────────────────           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 关键数学推导

这是这个实验最精妙的数学洞察！

```
定义总有效 token 数：N

Normal 的准确率：
Accᴺ = (正确的 tokens) / N
     = (C→C + C→W) / N
       ↑────┘    ↑────┘
       都对      原本对

Variant 的准确率：
Accⱽ = (正确的 tokens) / N
     = (C→C + W→C) / N
       ↑────┘    ↑────┘
       都对      变成对

准确率变化：
ΔAcc = Accⱽ - Accᴺ
     = [(C→C + W→C) / N] - [(C→C + C→W) / N]
     = (W→C - C→W) / N

═══════════════════════════════════════════════════════════════
🔑 关键公式：

         ΔAcc = (W→C - C→W) / N

这个公式说明：
• 准确率变化只取决于 W→C 和 C→W 的差！
• C→C 和 W→W 对 ΔAcc 没有贡献
• 如果 W→C ≈ C→W，则 ΔAcc ≈ 0
═══════════════════════════════════════════════════════════════
```

### 2.3 天平比喻

```
想象一个天平：

              ⚖️ ΔAcc
               │
      ┌────────┴────────┐
      │                 │
   W→C                C→W
 (加分项)           (减分项)
   👍                 👎


情况 1: W→C >> C→W (理想情况)
─────────────────────────────
      ⚖️
       │
    ┌──┴──┐
████│     │▌
 W→C│    C→W

天平左倾 → ΔAcc > 0 ✅
准确率提升！


情况 2: W→C ≈ C→W (本实验发现的情况!)
─────────────────────────────
      ⚖️
       │
   ┌───┴───┐
 ███│     │███
  W→C    C→W

天平平衡 → ΔAcc ≈ 0 ⚠️
大量活动但净效果为零！
```

## 二、基础知识：决策边界与 Margin

### 2.1 什么是 Margin？

```
Softmax 输出概率分布：
p = softmax(z) = [p₁, p₂, ..., p₄₀₉₆]

其中 pᵢ = exp(zᵢ) / Σⱼ exp(zⱼ)


找出 Top-2：
c₁ = argmax pᵢ    (概率最大的类，即预测结果)
c₂ = argmax pᵢ    (概率第二大的类)
     i≠c₁

Margin（决策边界距离）：
m = p_c₁ - p_c₂  ∈ [0, 1]


物理意义：
═══════════════════════════════════════════════════════════════
• m ≈ 0    → 前两名非常接近 → 决策不确定 → 容易改变
• m ≈ 0.5  → 有一定差距 → 中等确定性
• m ≈ 1    → 第一名遥遥领先 → 高度确定 → 很难改变
═══════════════════════════════════════════════════════════════
```

### 2.2 Margin 的几何意义

```
想象一个分类空间：

高维 Logit 空间 (简化为 2D)：

               Decision Boundary
                     │
    Class A          │          Class B
                     │
         ⭐          │            ☆
       (c₁=A)        │        (c₂=B)
                     │
         p_A         │          p_B
                     │

Margin = |p_A - p_B|


情况 1: 低 Margin (靠近边界)
─────────────────────────────────────
    A          │          B
               │
      ⭐      ☆│
      p_A≈0.51 │p_B≈0.49
               │
Margin ≈ 0.02  → 非常接近边界
               → 轻微扰动就会越界
               → 但可能只是在 A↔B 间跳动


情况 2: 高 Margin (远离边界)
─────────────────────────────────────
    A          │          B
               │
  ⭐          │              ☆
  p_A=0.95     │          p_B=0.02
               │
Margin = 0.93  → 远离边界
               → 需要巨大扰动才能越界
               → 预测很稳定
```

### 2.3 Top-k Overlap 的定义

```
对于每个 token，取 Top-k 候选：

Normal Speaker:
  Kᴺ = {c₁ᴺ, c₂ᴺ, ..., cₖᴺ}   (前 k 个最可能的类)

Variant Speaker:
  Kⱽ = {c₁ⱽ, c₂ⱽ, ..., cₖⱽ}

Top-k Overlap:
  overlap = |Kᴺ ∩ Kⱽ| / k


物理意义：
═══════════════════════════════════════════════════════════════
• overlap = 1.0  → 两个 Top-k 完全相同 → 非常稳定
• overlap = 0.6  → 有 60% 重叠 → 部分改变，但方向相似
• overlap = 0.0  → 完全不同 → 彻底洗牌
═══════════════════════════════════════════════════════════════


例子 (k=5):
─────────────────────────────────────────────────────────────
Normal Top-5:   [1234, 567, 89, 1001, 2345]
Variant Top-5:  [1234, 89, 1001, 3456, 4567]

交集: {1234, 89, 1001}  → 3 个
Overlap = 3/5 = 0.6  → 60% 重叠
```

## 四、洗牌 vs 方向性翻转

### 4.1 什么是"洗牌"？

```
洗牌翻转的特征：
═══════════════════════════════════════════════════════════════

Normal Top-5:   [A, B, C, D, E]
Variant Top-5:  [B, A, D, C, F]

观察：
• 预测改变了 (A → B)
• 但前几名都是相似的候选
• 如果真实答案是 Z，两次都错！

物理解释：
• 在决策空间中，在几个相似点之间徘徊
• 没有"朝目标方向"移动
• Top-k overlap 很低，但都不对


类比：
想象找宝藏，有一张模糊的地图

洗牌式搜索：
  第一次: "我觉得在 A 点！"
  第二次: "不对，应该在 B 点！"
  第三次: "也许是 C 点？"

  结果: 在 A/B/C 间来回跳，但宝藏其实在 Z 点
       → 虽然位置一直在变，但从未接近目标
```

### 4.2 什么是"方向性"翻转？

```
方向性翻转的特征：
═══════════════════════════════════════════════════════════════

Normal Top-5:   [A, B, C, D, E]    (真实答案: C)
Variant Top-5:  [C, B, A, D, E]

观察：
• 预测从 A → C
• 而 C 正是目标！
• Target 在 Top-k 中的排名提升了

物理解释：
• 在决策空间中，朝目标方向移动
• Speaker 信息提供了有用的引导
• Top-k overlap 高（方向一致性），且更接近目标


类比：
有方向性的搜索：
  第一次: "我觉得在北边"
  提示: "宝藏在东北方向" (speaker)
  第二次: "那应该在东北边！" → 更接近了！

  结果: 虽然还没找到，但在向正确方向移动
       → 有实质性进展
```

## 二、基础知识：Logits 空间几何

### 2.1 什么是 Logits？

```
模型的输出流程：

Token Embeddings (B, T, 512)
    ↓
Transformer Encoder
    ↓
Output Projection (B, T, 4096)  ← Logits
    ↓
Softmax
    ↓
Probabilities (B, T, 4096)
    ↓
argmax
    ↓
Prediction


Logits 是 softmax 之前的"原始分数"
z = [z₁, z₂, ..., z₄₀₉₆]  ∈ ℝ⁴⁰⁹⁶

每个 zᵢ 代表模型对第 i 个 token 的"支持程度"
```

### 2.2 Logits 差分 Δz

```
定义两次前向传播的 logits：

zᴺ = model(noisy, normal_speaker)     Normal
z⁰ = model(noisy, zero_speaker)       Zero

Logits 差分：
Δz = zᴺ - z⁰  ∈ ℝ⁴⁰⁹⁶


物理意义：
═══════════════════════════════════════════════════════════════
Δz 向量表示 speaker 注入对每个类别支持度的改变

如果 Δz₁₂₃₄ = +5   → speaker 让类别 1234 的分数增加 5
如果 Δz₅₆₇  = -3   → speaker 让类别 567 的分数减少 3
═══════════════════════════════════════════════════════════════
```

### 2.3 目标方向向量

```
问题：什么是"朝目标方向"？

假设：
• 真实标签: y = target
• Normal 预测的 top-1: ĉ₁ (可能对也可能错)
• Normal 预测的 top-2: ĉ₂


理想的 Δz 应该：
═══════════════════════════════════════════════════════════════
1. 增加 target 的分数
2. 相对于竞争者 (如 ĉ₂) 增加得更多

定义目标方向向量：
d = e_target - e_ĉ₂

其中 eᵢ 是 one-hot 向量（第 i 维为 1，其余为 0）


如果 Δz 与 d 对齐（同向），则：
• Δz 在 target 维度上的分量大
• 在 ĉ₂ 维度上的分量小或负
• → 有助于修正预测！
═══════════════════════════════════════════════════════════════
```

### 2.4 余弦相似度

```
测量 Δz 与目标方向 d 的对齐程度：

cos(θ) = ⟨Δz, d⟩ / (‖Δz‖ · ‖d‖)

其中：
• ⟨·,·⟩ 是内积
• ‖·‖ 是欧氏范数
• ‖d‖ = √2  (因为 d = e_t - e_c₂)


物理意义：
═══════════════════════════════════════════════════════════════
cos(θ) = +1   → Δz 完全对齐 d  → 最理想！
cos(θ) =  0   → Δz 垂直于 d   → 无效（正交）
cos(θ) = -1   → Δz 反向于 d   → 反效果！
═══════════════════════════════════════════════════════════════


几何图示：

      Target
        ⭐
        ↑
        │ d (目标方向)
        │
        │
    ────┼────→ ĉ₂
        │
        │
        ↓
      Other


情况 1: cos > 0 (好)
─────────────────────────
        ⭐
        ↑ d
        │ ↗ Δz (正向对齐)
        │↗
    ────┼────

→ Δz 指向 target 方向


情况 2: cos ≈ 0 (无效)
─────────────────────────
        ⭐
        ↑ d
        │
        │    Δz →
    ────┼────────

→ Δz 垂直，不朝 target


情况 3: cos < 0 (糟)
─────────────────────────
        ⭐
        ↑ d
        │
        │↘ Δz (反向)
    ────┼────
        ↓

→ Δz 远离 target！
```

### 2.5 Margin 变化 Δm

```
另一个度量：目标的相对优势变化

Margin (target vs ĉ₂):
m = z_target - z_ĉ₂


Margin 变化：
Δm = mᴺ - m⁰
   = (z_target^N - z_ĉ₂^N) - (z_target^0 - z_ĉ₂^0)
   = Δz_target - Δz_ĉ₂


物理意义：
═══════════════════════════════════════════════════════════════
Δm > 0  → target 的相对优势增加 ✅
Δm = 0  → 相对优势不变
Δm < 0  → target 的相对优势减少 ❌
═══════════════════════════════════════════════════════════════
```

## 四、完整机制解释

### 4.1 为什么 Δz 缺乏方向性？

```
可能的原因：

1️⃣ 注入位置太早
═══════════════════════════════════════════════════════════════
Speaker 只在输入层通过 Cross-Attention 注入
    ↓
经过多层 Transformer + LayerNorm
    ↓
Speaker 信号逐渐被"稀释"或"扭曲"
    ↓
到输出层时，方向性已经丢失


2️⃣ 注入方式不够强
═══════════════════════════════════════════════════════════════
Cross-Attention 是 residual connection:
  output = input + cross_attn(input, speaker)

如果 cross_attn 的幅度相对 input 太小
→ speaker 的影响被淹没


3️⃣ 训练目标不匹配
═══════════════════════════════════════════════════════════════
模型训练时只优化 cross-entropy loss
没有显式鼓励 Δz 朝目标方向

Loss 只关心"预测对不对"
不关心"speaker 如何影响预测"
```

### 4.2 为什么中 Margin 区仍有效果？

```
尽管整体 cos 偏负，但：

1️⃣ 变异性较大
═══════════════════════════════════════════════════════════════
在中 margin 区，不同 tokens 的 cos 值变化较大
有些 token 的 cos 是正的（虽然平均是负的）
这些正向的 tokens → W→C


2️⃣ 需要的改变量适中
═══════════════════════════════════════════════════════════════
低 margin: 太容易改变，随机扰动就够
高 margin: 太难改变，需要巨大力量

中 margin: 刚好需要"适度且有方向性"的推动
→ 当 Δz 有点方向性时，就能成功翻转


类比：
低 margin: 像推倒一根羽毛，随便吹一口气就行
中 margin: 像推倒一张牌，需要准确的轻推
高 margin: 像推墙，怎么推都推不动
```

## 三、未来改进方向

基于这5个commits的发现，有明确的改进方向：

### 方向 1: Margin-Aware Gating

```
动机：不同 margin 区需要不同的处理策略

方法：
═══════════════════════════════════════════════════════════════
gate = g(margin)  # margin-aware 门控函数

output = token_emb + gate * cross_attn(token_emb, speaker)


设计：
• 低 margin 区: gate ≈ 0.2  (抑制随机扰动)
• 中 margin 区: gate ≈ 1.0  (充分利用)
• 高 margin 区: gate ≈ 0.5  (保守使用)


期望效果：
• 减少低 margin 区的破坏性翻转（降低 C→W）
• 保留中 margin 区的修正性翻转（保持 W→C）
• 整体提升 W→C - C→W
```

### 方向 2: 多层注入

```
动机：单次前置注入被 LayerNorm 稀释

方法：
═══════════════════════════════════════════════════════════════
Input Layer:  cross_attn_1(token_emb, speaker)
    ↓
Encoder Layer 1
    ↓
Mid-Injection: cross_attn_2(hidden, speaker)
    ↓
Encoder Layer 2, 3
    ↓
Late-Injection: cross_attn_3(hidden, speaker)
    ↓
Output


期望效果：
• Speaker 信息在多个层次被使用
• 不容易被稀释
• 更容易保持方向性
```

### 方向 3: 方向性辅助损失

```
动机：显式鼓励 Δz 朝目标方向

方法：
═══════════════════════════════════════════════════════════════
主损失：
L_main = CrossEntropy(logits, target)

方向性损失：
L_dir = -cos(Δz, d)    # 鼓励 Δz 与目标方向对齐
      = -⟨Δz, e_target - e_c2⟩ / ‖Δz‖

或者：
L_margin = -Δm = -(Δz_target - Δz_c2)  # 鼓励 target margin 增大

总损失：
L = L_main + λ * L_dir


期望效果：
• 直接优化 Δz 的方向性
• 提升中 margin 区的 W→C 概率
• 减少低 margin 区的无效洗牌
```
