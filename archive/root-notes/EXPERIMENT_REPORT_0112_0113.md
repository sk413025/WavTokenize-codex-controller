# WavTokenizer LoRA 降噪實驗報告
## Exp 0112 / 0113 系列實驗總結

**日期**: 2026-01-14
**實驗目標**: 探索有效的 LoRA 訓練策略，使 WavTokenizer 具備降噪能力
**核心問題**: 如何讓模型在淺/中層有效學習去噪，而非深層過度補償？

---

## 一、背景與問題

### 1.1 Feature Map 分析發現

基於 `exp_1231_feature` 的分析結果：

| 層組 | 層 | 噪音敏感度 (cos_sim) | 特性 |
|------|-----|---------------------|------|
| input | L0 | - | 原始波形 |
| low_level | L1-L4 | 中等 | 聲學特徵 (音色、音高) |
| **mid_level** | **L5-L8** | **0.21-0.29 (最敏感)** | **噪音破壞最嚴重** |
| semantic | L9-L12 | 中等 | 語義編碼開始 |
| abstract | L13-L16 | 0.72-0.82 (最魯棒) | 高階語義，幾乎免疫噪音 |
| output | L17 | - | VQ 投影 |

### 1.2 核心矛盾

- **降噪擾動在中層 (L5-L8) 最大**
- **但 LoRA 訓練後深層變化最大** (深層 1.73 vs 淺層 1.43)
- **假說**: 淺層容量不足 → 深層被迫「硬補償」 → 導致 Train/Val Gap

## 二、實驗設計總覽

### 2.1 Exp 0112 系列

| 實驗 | 策略 | 核心思想 |
|------|------|---------|
| **Exp I** | 三區差異化 LR | 中層高 LR，淺層 L2 正則化 |
| **Exp J** | 單一 Adapter | 輕量化去噪，只訓練 8K 參數 |
| **Exp K** | 中間層監督 | 直接監督 L4/L8 的 MSE |

### 2.2 Exp 0113 系列

| 實驗 | 策略 | 核心思想 |
|------|------|---------|
| **Exp L** | 多位置 Adapter | 增加 Adapter 容量和覆蓋 |
| **Exp M** | Adapter + LoRA 混合 | 結合兩者優勢 |

### 2.3 Exp Test (容量瓶頸測試)

| 實驗 | 策略 | 核心思想 |
|------|------|---------|
| **shallow_r256** | 只訓練 L0-L4, rank=256 | 測試淺層容量下限 |
| **shallow_r512** | 只訓練 L0-L4, rank=512 | 測試容量是否瓶頸 |
| **shallow_r1024** | 只訓練 L0-L4, rank=1024 | 測試大容量效果 |

## 三、實驗結果

### 3.1 主要指標對比

| 實驗 | Best Val Acc | Best Val Loss | Train-Val Gap | 參數量 |
|------|-------------|---------------|---------------|--------|
| **Exp I** | **0.77%** | **1.13** | 1.47% | ~3.7M |
| Exp J | 0.35% | 4.68 | ~0% | ~8K |
| Exp K | 0.80% | 420.51 ⚠️ | 1.24% | ~3.7M |
| Exp L | 0.51% | 1.30 | 0.04% | ~70K |
| Exp M | 0.42% | 3.84 | -0.01% | ~200K |

### 3.2 排名分析

**精度排名** (越高越好):
```
Exp K (0.80%) ≈ Exp I (0.77%) > Exp L (0.51%) > Exp M (0.42%) > Exp J (0.35%)
```

**Loss 排名** (越低越好):
```
Exp I (1.13) < Exp L (1.30) < Exp M (3.84) < Exp J (4.68) << Exp K (420) ⚠️
```

**參數效率** (精度/參數量):
```
Exp I: 0.77% / 3.7M = 最佳數值表現
Exp L: 0.51% / 70K = 最佳參數效率
```

## 四、各實驗詳細分析

### 4.1 Exp I: 三區差異化訓練 ⭐ 目前最佳

**架構設計**:
```
L0-L4  (淺層): LR = 5e-6  + L2 = 0.01  → 穩定基礎特徵
L5-L8  (中層): LR = 2e-5  + L2 = 0     → ★ 重點學習區
L9-L17 (深層): LR = 1e-5  + L2 = 0     → 自由配合
```

**結果**:
- Best Val Acc: **0.77%**
- Best Val Loss: **1.13**
- Train-Val Gap: 1.47% (適度過擬合)

**結論**: 三區差異化 LR 有效，中層高 LR 對應其噪音敏感性。

---

### 4.2 Exp J: 單一 Adapter ❌ 失敗

**架構設計**:
```
[L0-L4] (凍結) → Adapter (8K 參數) → [L5-L17] (凍結)
                      ↓
              LayerNorm → Conv↓ → GELU → Conv↑ → Residual
```

**結果**:
- Best Val Acc: **0.35%** (最差)
- Best Val Loss: 4.68

**失敗原因**: 8K 參數容量太小，無法學習有效的去噪映射。

---

### 4.3 Exp K: 中間層監督 ⚠️ 有潛力但異常

**架構設計**:
```
Teacher (L4) ──── MSE Loss ──── Student (L4)
Teacher (L8) ──── MSE Loss ──── Student (L8)
Teacher (Out) ─── Final Loss ── Student (Out)

Total Loss = 0.5×Loss_L4 + 0.5×Loss_L8 + Loss_final
```

**結果**:
- Best Val Acc: **0.80%** (最高！)
- Best Val Loss: **420.51** ⚠️ (異常高)

**問題**: Loss 值異常高 (420 vs Exp I 的 1.13)，可能是中間層 MSE 尺度不匹配。

**結論**: 中間層監督思路正確，但需要修正 Loss 計算。

---

### 4.4 Exp L: 多位置 Adapter 📈 改善但不足

**架構設計**:
```
Adapter @ L1 (model[1] 後)
Adapter @ L4 (model[4] 後)
Adapter @ L7 (model[7] 後)
Adapter @ L10 (model[10] 後)

reduction_factor = 2 (比 Exp J 的 4 更大容量)
總參數: ~70K (Exp J 的 ~9 倍)
```

**結果**:
- Best Val Acc: **0.51%** (比 J 改善 46%)
- Best Val Loss: **1.30** (接近 Exp I)
- Train-Val Gap: 0.04% (欠擬合)

**結論**: 增加 Adapter 容量有效改善，但仍不如全層 LoRA。

---

### 4.5 Exp M: Adapter + LoRA 混合 ❌ 失敗

**架構設計**:
```
淺層 (L1, L4):  大 Adapter (reduction=2)
中層 (L7):      小 Adapter (reduction=4) + 小 LoRA (rank=32)
深層 (L10-L16): 微量 LoRA (rank=16)
```

**結果**:
- Best Val Acc: **0.42%** (比 L 還差)
- Best Val Loss: 3.84
- Train-Val Gap: -0.01% (反向)

**失敗原因**: Adapter 和 LoRA 互相干擾，混合訓練信號過於複雜。

---

## 五、Exp Test: 淺層容量瓶頸測試

### 5.1 實驗設計

**核心問題**: LoRA 容量是否是淺層學習不足的原因？

**設計**:
- 只訓練 L0-L4 (5 層)，凍結 L5-L17
- Loss: 只監督 L4 輸出的 MSE
- 測試 rank = 256 / 512 / 1024

### 5.2 已完成結果

| Rank | Best Val Loss | Final CosSim | 參數量 | 狀態 |
|------|--------------|--------------|--------|------|
| 256 | **51.79** | 0.210 | 116K | ✅ 完成 |
| 512 | (進行中) | - | 233K | 🔄 135/150 |
| 1024 | (待執行) | - | 466K | ⏳ 等待 |

### 5.3 預期結論

- **若 rank↑ → loss↓↓**: 容量是瓶頸，淺層需要更大 LoRA
- **若 rank↑ → loss 無改善**: 問題不在容量，可能是梯度流或架構問題

---

## 六、關鍵發現

### 6.1 成功的策略

1. **三區差異化 LR (Exp I)**: 目前最佳，中層高 LR 符合噪音敏感性分布
2. **中間層監督 (Exp K)**: 精度最高，但 Loss 計算需修正
3. **多位置 Adapter (Exp L)**: 輕量化的有效替代方案

### 6.2 失敗的策略

1. **單一 Adapter (Exp J)**: 容量太小 (8K) 無法學習
2. **混合策略 (Exp M)**: Adapter 和 LoRA 互相干擾

### 6.3 核心洞察

```
┌────────────────────────────────────────────────────────────┐
│ 1. 容量很重要: 8K → 0.35%, 70K → 0.51%, 3.7M → 0.77%      │
│ 2. 分層策略有效: 三區 LR 比統一 LR 更好                    │
│ 3. 中間層監督有潛力: 直接指導噪音敏感層                    │
│ 4. 混合不一定好: 策略組合可能互相干擾                      │
└────────────────────────────────────────────────────────────┘
```
