## 🎯 實驗配置

### 訓練參數
- **Epochs**: 300
- **Batch Size**: 8 (Gradient Accumulation: 2, Effective: 16)
- **Learning Rate**: 1e-4 → 1e-6 (cosine annealing with 10-epoch warmup)
- **Optimizer**: AdamW (weight_decay=0.01, grad_clip=1.0)
- **Intermediate Weight**: 0.03 (固定，相較 V1 的 0.5→0.25 warmdown)
- **數據來源**: data3 (filtered, 包含 plastic 材質)
- **特殊處理**: NaN 保護機制 + AMP (Automatic Mixed Precision)
- **Seed**: 42

### 模型架構
```python
# TeacherStudentRVQ 架構
- Teacher Encoder: WavTokenizer Encoder (Frozen, 18 Conv layers)
- Student Encoder: WavTokenizer Encoder + LoRA (r=256, α=512, dropout=0.2)
- RVQ: 4-layer Residual Vector Quantizer
  - Codebook Size: K=2048 per layer
  - EMA Update: decay=0.99
  - Dead-code Reset: threshold=2
  - Usage Penalty: 0.1 × log(cluster_size)
- Intermediate Supervision: L3(0.3) + L4(0.5) + L6(0.5) @ model indices [3,4,6]
```

### Loss 配置
```python
L_total = λ_quant · L_quant + λ_inter · L_inter + β_commit · L_commit

# V2 權重 (相較 V1 大幅降低 intermediate 權重)
λ_quant = 1.0     # 主損失：post-quant alignment (MSE)
λ_inter = 0.03    # 中間層監督 (cosine loss) — V1: 0.5
β_commit = 1.0    # RVQ commitment loss
λ_pre = 0.0       # Pre-quant loss (disabled)
```

### Curriculum Learning
```python
# Noise Ratio Schedule (200 epochs)
noise_ratio: 0.30 → 0.85 over epochs 0-200
- Phase increment: 0.0275 per 10 epochs
- Fixed at 0.85 for epochs 200-300
```

## 🔍 關鍵發現

### 1. V2 成功解決 V1 的崩潰問題 ✅

**V1 問題回顧**：
- Epoch 176-177: NaN 出現（L4 cosine loss zero-norm vectors）
- Epoch 192: 突然完全崩潰（entropy=-0.00, used=1/2048）
- Epoch 192+: 無法恢復，所有 token 映射到單一 code

**V2 改進措施**：
1. **降低 intermediate_weight**: 0.5 → **0.03** (減少 94% → 15% 佔比)
2. **NaN 保護機制**: 跳過 NaN batch，連續 NaN 時警告
3. **移除 warmdown**: 直接使用正確權重，避免補救式設計

**結果**：
- ✅ 全程 300 epochs 無 NaN
- ✅ Codebook 穩定（entropy 維持 8.5-9.2）
- ✅ 無突然崩潰現象

### 2. Top-10 Mass 仍未達 P3 標準 ⚠️

| Metric | V1 (Epoch 150) | V2 (Epoch 300) | Phase 3-2 (1000 steps) | P3 Target |
|--------|----------------|----------------|------------------------|-----------|
| **Entropy** | 9.03 | 9.01 | 9.03 | >6.5 ✅ |
| **Top-10 Mass** | 14.8% | **18.9%** | 15.8% | <15% ❌ |
| **Used Codes** | 1027/2048 | 1142/2048 | 1089/2048 | ≥205 ✅ |
| **Joint Div** | 0.961 | 0.975 | 0.992 | >0.7 ✅ |

**觀察**：
- V2 Top-10 Mass (18.9%) 比 V1 (14.8%) 和 Phase 3-2 (15.8%) 都**略高**
- 可能原因：Curriculum learning 高噪音階段（noise 0.85）導致部分 token 過度集中
- 建議：考慮調整 usage penalty 或 noise schedule

### 3. 與 V1 版本對比

| 項目 | V1 | V2 | 結果 |
|------|----|----|------|
| **Intermediate Weight** | 0.5 → 0.25 warmdown | 0.03 固定 | V2 更簡潔 ✅ |
| **Quant Loss 梯度佔比** | ~5% | ~80-90% | V2 主導正確 ✅ |
| **訓練穩定性** | Epoch 192 崩潰 | 全程穩定 | V2 大幅改善 ✅ |

| **P3 驗收** | PASS (Epoch 150) | FAIL (Top-10 18.9%) | V1 略優 ⚠️ |

**結論**：
- ✅ V2 解決了 V1 的致命崩潰問題
- ⚠️ V2 的 Top-10 Mass 略高於 V1 和 Phase 3-2，需進一步調優
- ✅ 整體而言，V2 是更穩定、可靠的版本

## 🐛 遇到的問題

### 1. NaN 崩潰問題 (V1 → V2 已解決) ✅

**V1 問題**：
- **時間點**: Epoch 176-177 開始出現 NaN
- **崩潰點**: Epoch 192 完全崩潰（entropy=-0.00, used=1/2048）
- **根本原因**:
  1. Cosine loss 中 `F.normalize()` 遇到 zero-norm vectors → NaN
  2. Intermediate weight 過高 (0.5，佔 total loss 94%) 壓制 quant loss 梯度
  3. Curriculum learning 高噪音階段（noise ~0.78）導致 L4 層輸出衰減

**V2 解決方案**：
- ✅ 降低 `intermediate_weight`: 0.5 → 0.03 (quant loss 梯度佔比提升至 80-90%)
- ✅ 實作 NaN 保護：跳過 NaN batch + 連續 NaN 警告
- ✅ 移除 warmdown 機制：直接使用正確權重
- ✅ 建議加入 `eps=1e-8` 到 cosine loss（未在 V2 實作，但無 NaN 發生）

**驗證結果**：
- ✅ V2 全程 300 epochs 無 NaN
- ✅ Codebook 持續健康（entropy 8.5-9.2）
- ✅ 無突然崩潰現象

### 2. Top-10 Mass 未達 P3 標準 (V2 待優化) ⚠️

**問題**：
- V2 Epoch 300: Top-10 Mass = 18.9% (目標: <15%)
- 相比 V1 Epoch 150: 14.8% (PASS P3)
- 相比 Phase 3-2 短期: 15.8% (PASS P3)

**可能原因**：
1. Curriculum learning 終點 (noise=0.85) 過高，導致 token 集中
2. Usage penalty (0.1) 可能不足以平衡高噪音階段的影響
3. 300 epochs 過長，導致 codebook 過度優化少數常用 codes

**建議後續調整**：
- [ ] 降低 `curriculum_end`: 0.85 → 0.75
- [ ] 提高 `usage_penalty`: 0.1 → 0.15 或 0.2

### 3. Validation Set 音質下降 (待調查) ⚠️

**現象**：
- Train set: PESQ +0.017, STOI +0.042 ✅

**可能原因**：
1. **Overfitting**: 模型過度擬合 training data noise pattern
2. **Val noise 不匹配**: Val set 的 noise 特性與 train set 差異過大
3. **Codebook 泛化不佳**: RVQ 學到的 codes 無法良好表達 val set 特徵

**建議調查方向**：
- [ ] 檢查 train/val split 的 noise distribution

## 💡 心得與下一步

### 學到的經驗

1. **Loss Weight Balance 至關重要** 🎯
   - Auxiliary loss（intermediate）不應超過 main loss（quant）的 20%
   - V1 的 intermediate weight=0.5 導致梯度失衡（94% vs 5%）
   - V2 的 0.03 權重讓 quant loss 主導（80-90%），訓練更穩定

2. **Numerical Stability 不可忽視** 🛡️
   - Cosine loss 的 `F.normalize()` 對 zero-norm vectors 不安全
   - 必須加入 `eps=1e-8` 或 NaN 保護機制
   - 即使機率很低，NaN 一旦發生會累積污染整個模型

3. **Curriculum Learning 需要細調** 📈
   - 高噪音階段（0.85）雖然增加難度，但可能導致 token collapse
   - V2 的 Top-10 Mass (18.9%) 高於短期實驗，可能與 curriculum 終點有關
   - 需要在 "挑戰性" 和 "codebook 多樣性" 之間取得平衡

4. **Early Stopping 的重要性** ⏸️
   - Best model 在 epoch 87（val loss=0.0644）
   - Final model（epoch 300）val loss=0.0674，反而略差
   - 300 epochs 可能過長，導致 overfitting

5. **驗收標準的多維度性** 📊
   - P2/P3 驗收不只看 entropy，Top-10 Mass 同樣關鍵
   - V2 通過 P2 但未達 P3，顯示仍有優化空間
   - 音訊品質（PESQ/STOI）是最終評判標準
