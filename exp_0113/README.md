# Exp 0113: 增強版 Adapter 去噪實驗

## 實驗背景

基於 Exp J (exp_0112_adapter) 的關鍵發現設計的後續實驗。

### Exp J 關鍵發現

| 指標 | Exp J (Adapter) | Exp I (LoRA) | 分析 |
|------|----------------|--------------|------|
| **Val Acc** | 0.35% | 0.77% | 數值差 ❌ |
| **Val Loss** | 4.68 | 1.13 | 數值差 ❌ |
| **聽感** | **更好** ✅ | 較差 | **關鍵發現！** |
| **Train-Val Gap** | 0.00% | 1.46% | 無過擬合 ✅ |
| **參數量** | 8,609 | 3,700,000 | 0.23% |

### 診斷結論

1. **Adapter 方向正確**：聽感更好說明 Adapter 產生更自然的特徵
2. **容量嚴重不足**：8K 參數無法學習複雜去噪模式
3. **單一位置不夠**：只在 L4 插入，無法涵蓋所有噪音敏感區域

## 實驗設計

### Exp L: 增強版多位置 Adapter

**策略**：增加 Adapter 數量和容量

```
┌─────────────────────────────────────────────────────────────────┐
│  WavTokenizer Encoder with Multi-Position Adapters              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  L0-L1: Conv layers (不插入)                                    │
│     ↓                                                            │
│  L2:   Downsample → [Adapter_L2: 64→32→64] ← 新增               │
│     ↓                                                            │
│  L3:   ResBlock                                                  │
│     ↓                                                            │
│  L4:   Downsample → [Adapter_L4: 128→64→128] ← 增強             │
│     ↓                                                            │
│  L5:   ResBlock                                                  │
│     ↓                                                            │
│  L6:   Downsample → [Adapter_L6: 256→64→256] ← 新增             │
│     ↓                                                            │
│  L7-L8: ResBlocks → [Adapter_L8: 256→64→256] ← 新增             │
│     ↓                                                            │
│  L9-L17: 深層 (不插入 Adapter)                                  │
│     ↓                                                            │
│  Quantizer → Tokens                                              │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

**配置**：
- Adapter 位置: L2, L4, L6, L8 (4 個位置)
- Bottleneck: input_dim → input_dim//2 → input_dim (比 Exp J 的 //4 更大)
- 預估參數: ~70K (Exp J 的 8 倍)

**預期**：
- 保持 Exp J 的聽感優勢
- 通過更大容量提升數值表現
- 覆蓋更多噪音敏感區域

---

### Exp M: Adapter + LoRA 混合

**策略**：結合 Adapter 的「聽感優勢」和 LoRA 的「數值表現」

```
┌─────────────────────────────────────────────────────────────────┐
│  Hybrid Adapter + LoRA Architecture                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │  淺層 (L0-L4): 大 Adapter + 無 LoRA                       │   │
│  │  ├─ Adapter_L2: 64 → 32 → 64                             │   │
│  │  └─ Adapter_L4: 128 → 64 → 128                           │   │
│  │  目的: 激進去噪，保持聽感優勢                            │   │
│  └──────────────────────────────────────────────────────────┘   │
│     ↓                                                            │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │  中層 (L5-L8): 小 Adapter + 小 LoRA (rank=32)            │   │
│  │  ├─ Adapter_L6: 256 → 64 → 256                           │   │
│  │  └─ LoRA on all Conv1d layers                             │   │
│  │  目的: 細化去噪，開始精細調整                            │   │
│  └──────────────────────────────────────────────────────────┘   │
│     ↓                                                            │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │  深層 (L9-L17): 無 Adapter + 微量 LoRA (rank=16)         │   │
│  │  └─ LoRA on all Conv1d layers                             │   │
│  │  目的: 輕微調整以適應上游修改                            │   │
│  └──────────────────────────────────────────────────────────┘   │
│     ↓                                                            │
│  Quantizer → Tokens                                              │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

**配置**：
- 淺層 Adapter: L2, L4 (較大 bottleneck)
- 中層 Adapter + LoRA: L6 Adapter + rank=32 LoRA
- 深層 LoRA only: rank=16
- 預估參數: ~200K-300K (LoRA 3.7M 的 5-8%)

**預期**：
- 超越 Baseline (1.06% Val Acc)
- 保持較小的 Train-Val Gap
- 結合兩種方法的優點
