## 3. 分析結果

### 3.1 問題 1: 原始 WavTokenizer 各層在看什麼？

**實驗方法**: 比較不同說話者 (boy1 vs girl2) 說同一句話時，各層 feature map 的相似度

**結果**:

| 層組 | 層 | Cosine Similarity | 解讀 |
|------|-----|-------------------|------|
| input | L0 | -0.011 | 完全不同（原始波形差異大） |
| low_level | L1-L4 | 0.021 (avg) | 幾乎無關（聲學特徵不同） |
| mid_level | L5-L8 | 0.068 (avg) | 略有相關 |
| semantic | L9-L12 | 0.117 (avg) | 開始收斂 |
| **abstract** | **L13-L16** | **0.433 (avg)** | **高度相似！** |
| output | L17 | 0.187 | 投影後略降 |

**Peak layers**:
- **L15: 0.672** (model.10.shortcut) - 最高 speaker-invariance
- **L13: 0.522** (model.10.block.1)

**視覺化**:

### 3.3 問題 3: Train/Val 是否看同樣特徵？

**狀態**: ✅ 已完成分析

**核心問題澄清**:
```
❌ 錯誤的問題: Train/Val 的 feature 數值分布是否相同？
   → 答案當然不同，因為 Val 放未見過的說話者是正確的設計

✓ 正確的問題: Train/Val 是否關注「同類型」的特徵？
   → 類比: 貓咪分類中，兩者都應該看「貓輪廓」而非一個看「背景沙發」
```

**實驗方法**:
1. 從 train/val cache 各取 20 個 noisy audio 樣本
2. 用原始 WavTokenizer 提取各層 feature map
3. 計算 Activation Pattern（哪些 channel 被激活的分布）
4. 比較 Train/Val 的 attention pattern 一致性 (Cosine Similarity)
5. 計算 CKA (Centered Kernel Alignment) 驗證表示相似度

**結果**:

| 指標 | 數值 | 解讀 |
|------|------|------|
| **Pattern Similarity** | **0.9907** | ✅ 極高！Train/Val 關注相同特徵 |
| CKA | 0.3209 | 中等（不同說話者，絕對值不同是正常的） |

**各層 Pattern Similarity**:

## 4. 發現的矛盾與診斷

### 4.1 觀察到的矛盾

```
┌─────────────────────────────────────────────────────────────┐
│ 理論預期 (降噪任務)            vs    實際觀察 (LoRA 變化)     │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│ 淺層應該修改最多                │  淺層變化較小 (1.56)       │
│ (因為噪音影響聲學特徵)          │                            │
│                                 │                            │
│ 深層應該變化最小                │  深層變化最大 (1.73) ❌     │
│ (因為語義已經魯棒)              │                            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 可能的解釋

**解釋 1: Loss 函數位置導致**

```
Loss 作用在 encoder output (L17 後)
    │
    ▼ 梯度反向傳播
L17 ← L16 ← L15 ← ... ← L0
 │
 └── 深層先收到強梯度，淺層梯度逐漸衰減

結論: 深層變化大可能只是因為「離 loss 近」，
      不代表是正確的學習策略
```

**解釋 2: 模型在「語義層硬補償」**

```
正確的降噪:
  noisy → [淺層修正噪音] → [深層保持語義]

可能發生的:
  noisy → [淺層沒怎麼動] → [深層強行適應 noisy 特徵]

這就像圖像分類模型「看背景沙發」而非「看貓輪廓」
  - 在訓練集上可以 work（記住了特定背景）
  - 在驗證集上失敗（背景不同就不認識）
```

### 4.3 這可能解釋 Train/Val gap

```
如果模型在深層「硬記」訓練集的噪音模式：
  - Train accuracy ↑ (因為記住了訓練集的 noisy→clean 映射)
  - Val accuracy 停滯 (因為驗證集的噪音模式不同)

這與我們觀察到的症狀吻合！
```

## 5. 之前「只開 8 層」失敗的重新解讀

### 5.1 原本的推理

```
觀察: critical_8 效果差 (8% vs 11%)
結論: 遺漏了「關鍵層」(model.7, model.10)
解法: 加入這些層 → critical_10
```

### 5.2 新的可能解釋

問題可能不是「遺漏關鍵層」，而是「分布偏移」：

```
當只修改 8 層時:

  Layer 0-7:  有 LoRA → feature 分布改變
       ↓
  Layer 8-17: 無 LoRA → 期待原始分布
       ↓
  分布不匹配 → 無法正確處理
```

**證據**: 提高 rank (256→512) 無法彌補，說明問題是「覆蓋範圍」而非「容量」

### 5.3 建議的策略

```
不是「開/關」的二元選擇，而是「程度差異」：

┌─────────────────────────────────────────────────────────────┐
│  淺層 (L0-L8):  高 rank (256) → 大幅修改（去除噪音）         │
│  深層 (L9-L17): 低 rank (16)  → 輕微調整（適應新分布）       │
└─────────────────────────────────────────────────────────────┘
```
