## 1. 研究背景與動機

### 1.1 問題描述

在使用 LoRA 微調 WavTokenizer encoder 進行音頻降噪任務時，觀察到：

| 現象 | 數值 |
|------|------|
| Train Accuracy | 持續上升 (3.22%) |
| Val Accuracy | 停滯不前 (1.06%) |
| **Train-Val Gap** | **2.33%** (過擬合徵兆) |

### 1.2 核心問題

> **模型是否「學歪了」？**
> 
> 類比圖像分類任務：
> - 正常模型: Feature map 在「貓的輪廓」發光 → 正確分類
> - 學歪模型: Feature map 在「背景沙發」發光 → 過擬合，無法泛化

### 1.3 研究方法

透過 Feature Map 分析，回答四個關鍵問題：

| # | 問題 | 分析方法 |
|---|------|----------|
| 1 | WavTokenizer 各層在看什麼？ | Speaker Invariance 分析 |
| 2 | LoRA 訓練後哪些層變化最大？ | Weight Change 分析 |
| 3 | Train/Val 是否關注相同特徵？ | Attention Pattern 分析 |
| 4 | 各層對噪音的敏感度如何？ | Noise Sensitivity 分析 |

## 3. Feature Map 分析實驗

### 3.1 分析一：Speaker Invariance（說話者不變性）

**實驗方法**: 比較不同說話者說同一句話時，各層 feature map 的 cosine similarity

**實驗假設**:
- 淺層提取聲學特徵 → 不同說話者差異大，similarity 低
- 深層提取語義特徵 → 同一內容趨於一致，similarity 高

#### 3.1.1 初步實驗 (2 位說話者)

比較 boy1 vs girl2 說同一句話：

| 層組 | 平均 Cosine Similarity | 解讀 |
|------|------------------------|------|
| Input (L0) | -0.011 | 完全不同 |
| Low-Level (L1-L4) | 0.017 | 幾乎無關 |
| Mid-Level (L5-L8) | 0.073 | 略有相關 |
| Semantic (L9-L12) | 0.118 | 開始收斂 |
| **Abstract (L13-L16)** | **0.427** | **高度相似** |
| Output (L17) | 0.187 | 投影後略降 |

> **📊 圖表位置**: `exp_1231_feature/outputs/speaker_invariance_summary.png`

#### 3.1.2 多語者驗證 (12 位說話者) ⭐ 新增

為了確保結論的統計顯著性，擴大到 12 位說話者進行驗證：

**實驗規模**:
| 項目 | 數值 |
|------|------|
| 說話者數 | 12 人 (6 男 + 6 女) |
| 每人句數 | 5 句 |
| 跨性別比較組數 | 180 組 |
| 同性別比較組數 | 150 組 (男 75 + 女 75) |
| **總比較組數** | **330 組** |

> **📊 圖表位置**: `exp_1231_feature/outputs/multi_speaker_analysis.png`

#### 圖表解讀指南

**Cosine Similarity 的意義（重要！）**:

| Cos 值範圍 | 意義 | 在本實驗中的解讀 |
|-----------|------|----------------|
| **+1** | 完全相同 | 不同說話者說同句話的特徵「完全一樣」→ 純內容特徵 |
| **0.5-0.7** | 高度相似 | 不同說話者說同句話的特徵「很相似」→ **語義特徵為主** ✅ |
| **0** | 不相關 | 不同說話者說同句話的特徵「無關」→ 音色特徵為主 |
| **負值** | 反向 | 不同說話者說同句話的特徵「相反」→ 對抗性特徵 |

**本實驗設定**: 比較不同說話者說**同一句話**
- **Cos 高** → 該層關注「內容語義」（好！對降噪魯棒）
- **Cos 低** → 該層關注「說話者音色」（壞！容易受噪音干擾）

### 3.3 分析三：Train/Val Attention Consistency（注意力一致性）

**實驗方法**: 
1. 從 train/val 各取 20 個樣本
2. 計算各層的 channel activation pattern
3. 比較 Train/Val 的 pattern similarity

**核心問題**:
> Train 和 Val 是否關注「相同類型」的特徵？  
> 還是一個在看「貓輪廓」，一個在看「背景沙發」？

#### 結果

| 指標 | 數值 | 解讀 |
|------|------|------|
| **Pattern Similarity** | **0.9907** | ✅ 極高！關注相同特徵 |

### 4.5 關鍵發現與深度分析

#### 4.5.1 為什麼 LoRA 會去動深層而不是淺層？

**答案：梯度反向傳播的「地理位置」效應**

```
┌─────────────────────────────────────────────────────────────────────┐
│                    梯度反向傳播機制                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  Loss (計算位置：最後一層)                                          │
│    ↑                                                                 │
│    │ 梯度 = 100 ← 離 loss 近，梯度大                                 │
│    ├─ L17 (Output)      相對變化: 1.46                              │
│    ├─ L16 (Abstract)    相對變化: 1.89                              │
│    ├─ L14 (Abstract)    相對變化: 1.89                              │
│    ├─ L13 (Abstract)    相對變化: 1.97 ★ 最大                       │
│    │                                                                 │
│    │ 梯度逐漸衰減（經過多層反向傳播）                                │
│    │                                                                 │
│    ├─ L8 (Mid-Level)    相對變化: 1.54                              │
│    ├─ L4 (Low-Level)    相對變化: 1.64                              │
│    └─ L0 (Input)        相對變化: 1.43 ← 離 loss 遠，梯度小         │
│                                                                      │
│  權重更新公式: W_new = W_old - LR × ∇Loss                           │
│  → 梯度大 → 權重變化大                                              │
│  → 深層收到大梯度 → LoRA 權重變化大                                 │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

**結論**: LoRA 動深層不是「選擇」，是梯度的物理特性！

#### 4.5.2 為什麼凍結深層反而結果更差？

這是最反直覺的發現！深層明明對噪音魯棒（Cos = 0.72），為什麼凍結後 Acc 崩潰？

**答案：特徵分布不匹配（Feature Distribution Mismatch）**

```
┌─────────────────────────────────────────────────────────────────────┐
│              Baseline 訓練 (深層可動)                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. 淺層 LoRA 學習（梯度小，學習不足）:                             │
│     Noisy Feature → 「有點」去噪 → 特徵分布輕微改變                 │
│       ↓                                                              │
│  2. 深層收到「變化後」的特徵（分布已改變 10%）:                     │
│     深層 LoRA: 「咦，特徵分布變了？我調整一下理解方式」             │
│     → 深層權重大幅調整以「適應」新的特徵分布                        │
│       ↓                                                              │
│  3. 結果:                                                            │
│     雖然淺層去噪不足，但深層「硬補償」→ 勉強能用                    │
│     但這種補償是「死記訓練集」→ 過擬合 (Gap = 2.33%)                │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────┐
│              Exp B (凍結深層 L9-L17)                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. 淺層 LoRA 學習（同樣梯度小，同樣學習不足）:                     │
│     Noisy Feature → 「有點」去噪 → 特徵分布輕微改變                 │
│       ↓                                                              │
│  2. 深層被凍結（還用原始權重）:                                     │
│     深層: 「我還用『舊知識』理解特徵」                               │
│     但淺層特徵分布已經變了 → 深層讀不懂 ❌                          │
│       ↓                                                              │
│  3. 結果:                                                            │
│     淺層說「新語言」，深層聽不懂 → 完全錯亂                          │
│     Val Acc 崩潰至 0.51% (僅 baseline 的 48%)                        │
│                                                                      │
│  比喻:                                                               │
│  ─────                                                               │
│  淺層改善聽力 → 寫出更清晰的草稿                                    │
│  但深層語義專家「被凍結」→ 還用舊方法理解新草稿                     │
│  → 完全讀不懂 → 翻譯全錯                                            │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

#### 4.5.3 核心洞察：症狀 vs 病因

```
┌─────────────────────────────────────────────────────────────────────┐
│                         症狀 vs 病因分析                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ❌ 錯誤診斷: 深層變化大 → 深層是問題                               │
│  ✅ 正確診斷: 深層變化大只是「症狀」                                │
│                                                                      │
│  ─────────────────────────────────────────────────────              │
│                                                                      │
│  真正問題（病因）:                                                   │
│  ───────────────                                                     │
│  1. 淺層學習不足（梯度小 → 權重變化小 → 去噪能力弱）               │
│  2. 淺層「半吊子去噪」→ 特徵分布改變但不徹底                       │
│  3. 深層被迫「硬補償」淺層的不足                                    │
│  4. 深層補償方式 = 死記訓練集噪音模式 → 過擬合                     │
│                                                                      │
│  ─────────────────────────────────────────────────────              │
│                                                                      │
│  為什麼凍結深層無效？                                                │
│  ───────────────────                                                 │
│  • 凍結深層 = 治標不治本                                            │
│  • 淺層學習依然不足（病因未解決）                                   │
│  • 深層無法適應淺層的變化 → 反而更糟                               │
│                                                                      │
│  ─────────────────────────────────────────────────────              │
│                                                                      │
│  正確方向:                                                           │
│  ─────────                                                           │
│  • 不是「限制深層」→ 而是「強化淺層」                               │
│  • 不是「凍結深層」→ 而是「協同訓練」                               │
│  • 深層需要「輕微調整」以適應淺層的正確修正                         │
│                                                                      │
│  實現方法:                                                           │
│  • 差異化學習率: 淺層 10x，深層 1x                                  │
│  • 差異化 Rank: 淺層 256，深層 32                                   │
│  • L2 正則化: 懲罰深層過度變化                                      │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 5. 差異化訓練策略

### 5.1 策略設計

基於分析結論，設計三種差異化訓練策略：

| 實驗 | 策略 | 原理 |
|------|------|------|
| **Exp F** | 差異化學習率 | 淺層 LR=1e-4 (10x)，深層 LR=1e-5 (1x) |
| **Exp G** | 差異化 Rank | 淺層 rank=256，深層 rank=32 |
| **Exp H** | L2 正則化 | 對深層權重變化施加懲罰 |

### 5.2 Exp F: 差異化學習率

**配置**:
```
淺層 (L0-L8):  LR = 1e-4  (基準的 10 倍)
深層 (L9-L17): LR = 1e-5  (基準)
```

**預期**: 強制淺層多學習，限制深層變化

### 5.3 Exp G: 差異化 Rank

**配置**:
```
淺層 (L0-L8):  Rank = 256, Alpha = 512
深層 (L9-L17): Rank = 32,  Alpha = 64
```

**預期**: 淺層有足夠容量學習複雜去噪，深層容量受限只能做簡單微調

### 5.4 Exp H: L2 正則化

**配置**:
```
全層訓練，對深層施加額外懲罰
淺層 (L0-L8):  Rank = 256, L2 weight = 0
深層 (L9-L17): Rank = 256, L2 weight = 0.1

L2 損失計算:
  L2_loss = ||W_current - W_initial||²
  Total_loss = Task_loss + λ × L2_loss
```

**預期**: 允許深層調整，但懲罰過度變化

### 7.4 核心問題的重新定義

```
┌─────────────────────────────────────────────────────────────────────┐
│                      問題重新定義                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ❌ 原問題定義 (錯誤):                                              │
│     「深層變化過大導致過擬合，需要限制深層」                        │
│                                                                      │
│  ✅ 正確問題定義:                                                    │
│     「淺/中層無法有效去噪，深層被迫補償，補償方式不泛化」           │
│                                                                      │
│  差異:                                                               │
│  ─────                                                               │
│  • 原定義: 深層是「罪魁禍首」→ 限制它                              │
│  • 新定義: 深層是「受害者」→ 需要更好的上游支持                    │
│                                                                      │
│  為什麼淺層學不好？                                                  │
│  ─────────────────                                                   │
│  1. 梯度衰減: 淺層收到的梯度只有深層的 ~10%                        │
│  2. 任務複雜: 去噪需要學習「噪音模式識別」這是高度非線性任務       │
│  3. 容量不足: 淺層 channel 數少 (32-64)，難以建模複雜噪音          │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 7.5 為何差異化 LR 只有部分效果？

```
Exp F 結果分析:
─────────────

配置: 淺層 LR = 1e-4 (10x), 深層 LR = 1e-5 (1x)

結果:
• Val Acc: 1.06% → 0.80% (下降 25%)
• Gap: 2.33% → 1.52% (改善 35%)

解讀:
─────
✅ Gap 改善: 淺層學更多 → 深層補償需求減少 → 過擬合減輕
❌ Acc 下降: 淺層 LR 過高 → 學習不穩定 → 整體性能下降

問題:
─────
10x LR 對淺層來說「過於激進」
淺層需要的不是「暴力提升 LR」而是「更好的訓練信號」
```

### 7.6 根本性問題：任務本身的挑戰

```
┌─────────────────────────────────────────────────────────────────────┐
│                   降噪任務的根本挑戰                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  WavTokenizer 的設計目的:                                           │
│  ─────────────────────────                                          │
│  • 將乾淨音頻編碼成離散 token                                       │
│  • 訓練時只見過乾淨音頻                                             │

│  • 輸入: 帶噪音頻                                                    │

│                                                                      │
│  LoRA 的能力限制:                                                    │
│  ────────────────                                                    │
│  • LoRA 是「輕量微調」，只調整 ~1% 的參數                           │
│  • 適合: 風格遷移、領域適應                                         │
```

### 8.3 未來研究方向

基於本系列實驗的發現，建議以下研究方向:

#### 方向 A: 改變微調方法

| 方法 | 原理 | 預期效果 |
|------|------|----------|
| **Full Fine-tuning** | 調整所有參數 | 更大的學習容量 |
| **Adapter Tuning** | 插入額外層 | 針對性添加去噪模組 |
| **中層聚焦 LoRA** | 只在 L5-L8 加 LoRA | 精準定位噪音敏感層 |

#### 方向 B: 改變架構

```
┌─────────────────────────────────────────────────────────────────────┐
│                      架構改進建議                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  Option 1: 前置去噪模組                                              │
│  ──────────────────────                                              │
│  Noisy Audio → [去噪網路] → Clean-ish Audio → WavTokenizer          │
│                                                                      │
│  優點: WavTokenizer 維持原始能力，去噪獨立學習                      │
│  缺點: 需要額外網路，推理成本增加                                   │
│                                                                      │
│  Option 2: 多任務學習                                                │
│  ────────────────────                                                │
│  同時訓練 tokenization + denoising，共享表示                        │
│                                                                      │
│  Option 3: 對比學習                                                  │
│  ────────────────                                                    │
│  讓模型學習「noisy 和 clean 應該映射到相同 token」                  │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

#### 方向 C: 改變訓練策略

| 策略 | 說明 | 針對問題 |
|------|------|----------|
| **更大數據集** | 增加訓練數據多樣性 | 可能改善泛化 |
| **課程學習** | 從低噪音逐漸增加到高噪音 | 漸進式學習去噪 |
| **Auxiliary Loss** | 在中層添加額外的去噪監督信號 | 直接指導淺層學習 |
