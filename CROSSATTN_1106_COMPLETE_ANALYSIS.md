#### 方案 A：Gated 模型 (Margin-Aware Gating)

```
核心思想：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
根据 token 的 margin 动态调整 speaker 注入强度

实现：
  output = token_emb + gate(margin) * cross_attn(token, speaker)
                        ↑
                    动态门控函数

期望效果：
  • 低 margin 区: gate ≈ 0.2  → 抑制破坏性翻转 (降低 C→W)
  • 中 margin 区: gate ≈ 1.0  → 充分利用修正性翻转 (保持 W→C)
  • 高 margin 区: gate ≈ 0.5  → 保守使用
```

#### 方案 B：Deep 模型 (多层注入)

```
核心思想：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
在多个 Transformer 层注入 speaker 信息，避免被稀释

实现：
  Input Layer:  cross_attn_1(token_emb, speaker)
      ↓
  Encoder Layer 1
      ↓
  Layer 1:      cross_attn_2(hidden, speaker)  ← 新增
      ↓
  Encoder Layer 2, 3
      ↓
  Layer 3:      cross_attn_3(hidden, speaker)  ← 新增
      ↓
  Output

期望效果：
  • Speaker 信号不容易被 LayerNorm 稀释
  • 更容易保持方向性
  • 每层都能访问 speaker 信息
```

### 九、未来方向建议

```
╔═══════════════════════════════════════════════════════════════╗
║              基于实验结果的改进建议                           ║
╠═══════════════════════════════════════════════════════════════╣
║                                                               ║
║  🥇 优先级 1: Gated + 方向性辅助损失                          ║
║  ═══════════════════════════════════════════════════         ║
║  Gated 已证明有效，但方向性仍是随机 (cos ≈ 0)                ║
║                                                               ║
║  实现：                                                       ║
║    L = L_CE + λ * L_dir                                      ║
║    其中 L_dir = -cos(Δz, d)                                  ║
║                                                               ║
║  期望：                                                       ║
║    • 保留 Gated 的区域选择性                                 ║
║    • 增加 Δz 朝目标方向的分量                                ║
║    • 从 cos ≈ 0 提升到 cos > 0.1                            ║
║    • 预期 ΔAcc 从 -9.8pp 提升到 -12~-15pp                    ║
║                                                               ║
║                                                               ║
║  🥈 优先级 2: 自适应门控函数                                  ║
║  ═══════════════════════════════════════════════════         ║
║  当前 gate(margin) 可能是固定映射，可以改为可学习             ║
║                                                               ║
║  实现：                                                       ║
║    gate = σ(MLP([margin, cos(Δz, d_est)]))                  ║
║                                                               ║
║  优势：                                                       ║
║    • 根据训练动态调整门控策略                                 ║
║    • 更精细的区域控制                                         ║
║                                                               ║
║                                                               ║
║  🥉 优先级 3: Gated + Deep 组合                               ║
║  ═══════════════════════════════════════════════════         ║
║  虽然 Deep 单独表现不佳，但可能与 Gated 互补                  ║
║                                                               ║
║  实现：                                                       ║
║    • Layer 0: cross_attn + margin-aware gate                 ║
║    • Layer 1: cross_attn + margin-aware gate                 ║
║    • Layer 3: cross_attn + margin-aware gate                 ║
║                                                               ║
║  预期：                                                       ║
║    • 每层都有选择性控制                                       ║
║    • 更强的信号 + 更好的控制                                  ║
║                                                               ║
╚═══════════════════════════════════════════════════════════════╝
```
