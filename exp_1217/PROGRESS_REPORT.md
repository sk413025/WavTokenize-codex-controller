## 一、研究目的

### 1.1 背景

WavTokenizer 是一種音訊 token 化模型，採用 VQ-VAE 架構將連續音訊信號轉換為離散 token 序列。本研究探索使用 **Teacher-Student 知識蒸餾架構** 進行 **zero-shot denoising**，目標是讓 Student 模型接收噪聲音訊後，能產生與 Teacher 模型處理乾淨音訊時相同的 token 輸出。

### 1.2 研究問題

基於先前實驗 (Exp35-37) 的觀察，我們提出以下研究問題：

1. **Loss 函數組合**: Feature Loss、Triplet Loss、CE Loss 三種損失函數如何影響模型性能？
2. **LoRA 配置優化**: 微調層數與 rank 參數如何影響學習效果？
3. **資料品質影響**: 訓練資料中的 Clean→Clean 樣本是否影響去噪學習？

### 1.3 實驗目標

- 透過消融實驗確定最佳 Loss 函數組合
- 探索 LoRA 層數與容量的最佳配置
- 識別並修復資料問題，提升模型性能

**訓練流程**:
1. Teacher (凍結) 接收乾淨音訊，產生目標 token
2. Student (LoRA 可訓練) 接收噪聲音訊，嘗試預測相同 token
3. 透過多種 Loss 函數計算差異並反向傳播

### 2.2 Loss 函數設計

本實驗設計三種損失函數：

#### 2.2.1 Feature Loss (MSE)

**原理**: 最小化 Teacher 與 Student 編碼器輸出的均方誤差

```
L_feature = (1/N) * Σ ||f_teacher(x_clean) - f_student(x_noisy)||²
```

**直觀解釋**:
- 計算 Teacher 編碼器輸出 `f_teacher(x_clean)` 與 Student 編碼器輸出 `f_student(x_noisy)` 之間的歐氏距離平方
- 對所有 N 個時間步取平均

**作用**:
- 保持特徵空間的連續性
- 確保相似音訊產生相近的特徵表示
- 有利於維持音質

**示意圖**:
```
Teacher: Clean Audio → Encoder → [f_t1, f_t2, ..., f_tN]
                                        ↓ MSE Loss
Student: Noisy Audio → Encoder → [f_s1, f_s2, ..., f_sN]
```

---

#### 2.2.2 Triplet Loss

**原理**: 確保 Student 預測的特徵更接近正確的 Teacher 特徵，而遠離錯誤的特徵

```
L_triplet = max(0, d(anchor, positive) - d(anchor, negative) + margin)

其中:
  - anchor   = Student 特徵 f_s
  - positive = 對應位置的 Teacher 特徵 f_t_pos (正樣本)
  - negative = 不對應位置的 Teacher 特徵 f_t_neg (負樣本)
  - d(a,b)   = ||a - b||² (歐氏距離平方)
  - margin   = 0.2 (安全邊界)
```

**直觀解釋**:
- 希望 Student 特徵到正確 Teacher 特徵的距離 < 到錯誤 Teacher 特徵的距離
- margin 確保兩者有足夠的差距

**作用**:
- 強化 token 鄰域關係的學習
- 提升預測準確率
- 但單獨使用會導致訓練崩潰

**示意圖**:
```
                    f_t_neg (負樣本)
                        ↑
                        | d_neg (應該較遠)
                        |
    f_s (anchor) -------|
        ↓
        | d_pos (應該較近)
        ↓
    f_t_pos (正樣本)

目標: d_pos + margin < d_neg
```

---

#### 2.2.3 Cross-Entropy Loss (CE)

**原理**: 直接優化 token 預測的分類準確率

```
L_CE = -Σ y_i * log(ŷ_i)

其中:
  - y_i  = Teacher 的 one-hot token 標籤
  - ŷ_i  = Student 對每個 token 的預測機率
```

**直觀解釋**:
- 將 token 預測視為 2048 類別的分類問題
- 最小化 Student 預測與 Teacher 目標之間的交叉熵

**作用**:
- 直接優化 token 預測
- 可能導致「投機行為」(speculation)，模型傾向預測少數高頻 token
- 犧牲特徵連續性以換取準確率

**示意圖**:
```
Teacher Token: 1542
              ↓
Student 預測機率分布:
[0.01, 0.02, ..., 0.85, ..., 0.01]  ← 希望 index=1542 機率最高
                   ↑
               index 1542
```

### 2.3 LoRA 微調策略

使用 **Low-Rank Adaptation (LoRA)** 進行參數高效微調：

| 配置 | 層數 | Rank | 參數量 | 說明 |
|------|------|------|--------|------|
| all_18 | 18層 | 128 | ~6.8M | 基準配置 |
| all_18 | 18層 | 256 | ~13.6M | 高容量 |
| critical_8 | 8層 | 256 | ~6.0M | 關鍵層策略 |
| critical_8 | 8層 | 512 | ~12.1M | 最高容量 |

**關鍵層選擇** (critical_8):
- 輸入投影層
- 各 downsampling block 主卷積
- 部分 Residual block
- 輸出投影層

#### 3.1.5 CE Loss 造成的 Mode Collapse 現象

**圖 1: Exp42 Code Distribution (三者平衡)**

![Exp42 Code Distribution](probe_exp42_balanced_dist.png)

**觀察**:
- **橙色 (Teacher Target)**: 分布相對均勻，覆蓋 0-2000 token 範圍
- **藍色 (Student Pred)**: 嚴重集中在 1700-1850 區間，出現明顯尖峰
- **診斷**: 發生 **Mode Collapse**，Student 傾向預測少數高頻 token

---

**圖 2: Exp43 Code Distribution (CE + Triplet)**

![Exp43 Code Distribution](probe_exp43_ce_triplet_dist.png)

**觀察**:
- **橙色 (Teacher Target)**: 分布均勻
- **藍色 (Student Pred)**: 出現多個離散尖峰 (約 250, 850, 950, 1050, 1250, 1350, 1500, 1700)
- **診斷**: **多模態 Collapse**，Student 學會「投機」預測特定 token

---

**圖像分析結論**:

CE Loss 導致模型出現「投機行為」(Speculation):
1. 模型發現預測少數高頻 token 能快速降低 CE Loss
2. 犧牲分布多樣性，換取表面準確率
3. 結果: 高 Accuracy 但高 Distance，音質差

#### 3.1.4 Triplet Loss 的穩定性問題

**Exp47 (純 Triplet) 失敗案例**:
- 13 epochs 後出現 NaN，訓練崩潰
- Accuracy 從 0.4% 急降至 0.07%
- Distance 暴漲至 7.08

**原因分析**:
- Triplet Loss 僅優化相對距離，缺乏絕對錨定
- 無 Feature Loss 穩定特徵空間，導致 collapse
- **結論**: Triplet Loss 必須配合 Feature Loss 使用

```
│   - 深層: Token 預測                                      │
│                                                          │
│   結論: 跳過中間層導致語義理解能力不足                    │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 3.3 資料問題發現與修復

#### 3.3.1 問題描述

分析 `train_cache.pt` 發現 **28% 樣本為 Clean→Clean 配對**:

| 類型 | 訓練集 | 驗證集 | 比例 |
|------|--------|--------|------|
| Noisy→Clean | 10,368 | 1,728 | 72% |
| **Clean→Clean** | 4,032 | 864 | **28%** |
| 總計 | 14,400 | 2,592 | 100% |

**問題影響**:
- Student 模型在 28% 樣本中接收到乾淨音訊
- 稀釋了 denoising 學習信號
- 模型部分學會「直接複製」而非「去噪」

#### 3.3.2 修復方案

修改 `data_aligned.py`，新增 Clean→Clean 過濾功能：

```python
def _is_clean_to_clean(self, sample):
    """判斷是否為 clean→clean 樣本"""
    noisy_path = sample.get('noisy_path', '')
    clean_path = sample.get('clean_path', '')

    # 方法1: 路徑相同
    if noisy_path == clean_path:
        return True

    # 方法2: noisy_path 包含 '_clean_'
    if '_clean_' in noisy_path:
        return True

    return False
```

**修復後資料量**:
- Train: 14,400 → 10,368 (過濾 4,032 筆)
- Val: 2,592 → 1,728 (過濾 864 筆)

## 五、討論

### 5.1 CE Loss 的問題

**觀察**: 使用 CE Loss 的實驗 (Exp40-43) 皆出現:
- Code distribution 的 Mode Collapse (圖 1, 2)
- Distance 指標惡化
- 準確率與品質的 trade-off

**解釋**:
CE Loss 直接優化 token 分類，導致模型學會「走捷徑」—預測少數高頻 token 來最小化 loss。這種「投機行為」提高了表面準確率，但破壞了 codebook 空間的語義結構。

**建議**:
在音訊 token 化任務中，應避免使用 CE Loss，改用 Feature Loss + Triplet Loss 的組合。

### 5.2 Triplet Loss 的雙面性

**優點**:
- 顯著提升準確率 (Exp35: 15.70% vs Exp36: 10.29%)
- 強化 token 鄰域關係學習

**缺點**:
- 單獨使用會導致訓練崩潰 (Exp47)
- 必須配合 Feature Loss 穩定訓練

**結論**: Triplet Loss 是提升性能的關鍵，但需要 Feature Loss 作為「錨定」。

### 5.3 LoRA 配置的影響

**發現**:
- 層數選擇比 rank 大小更重要
- 8 層策略失敗，說明 denoising 任務需要全編碼器參與
- Rank 128 已足夠，更高容量未帶來提升

**解釋**:
音訊去噪是全局任務，需要所有編碼層協同工作。只訓練關鍵層無法捕捉完整的噪聲-乾淨映射。

### 5.4 資料品質的重要性

**發現**: 28% Clean→Clean 樣本稀釋了學習信號

**影響**:
- Student 部分時間在學習「恆等映射」而非「去噪映射」
- 可能解釋為何 Phase 1 實驗無法超越 Exp35

**修復後期望**: Exp48 使用過濾後資料，預期性能提升。

## 六、結論

### 6.1 主要發現

1. **Loss 組合**: Feature Loss + Triplet Loss (無 CE) 是最佳組合
   - Feature Loss 穩定訓練並維持音質
   - Triplet Loss 提升預測準確率
   - CE Loss 導致 Mode Collapse，應避免使用

2. **LoRA 配置**: 全 18 層、rank=128 是最佳配置
   - 層數比 rank 更重要
   - 8 層策略不適用於去噪任務

3. **資料品質**: Clean→Clean 樣本會稀釋學習信號
   - 已修復，過濾後訓練集從 14,400 降至 10,368

### 6.2 最佳配置

| 參數類別 | 配置 | 說明 |
|----------|------|------|
| Loss | F=1.0, T=1.0, CE=0.0 | 經實驗驗證最佳 |
| LoRA 層數 | all_18 | 8層不足 |
| LoRA rank | 128 | 更高無顯著幫助 |
| 資料 | 過濾 Clean→Clean | 提升學習效率 |

### 6.3 後續工作

2. **Triplet 權重實驗**: 測試 T=1.5 或 T=2.0 是否進一步提升
3. **更長訓練**: 探索 1000+ epochs 的效果
