# Phase 3-3 結果分論：Hot-Code Branching / Codebook Split

日期：2026-02-05  
範圍：`exp_0128/phase3-3`（Exp7a/7b/7c/7d）  
依據：`PLAN.md` / `SPEC.md` / `ACCEPTANCE.md` + `PROGRESS.md`

---

## 1) 目標與核心假設（H1/H2）

- **H1（容量不足 / 模式混合）**：layer0 的 hot code 其實對應多模態 cluster（例如 speech/noise/silence），但因 collapse 或容量不足被迫塞進同一個 code。  
  - 預期：split 後 **多個 child 會被同時使用**（P1），且在 noise proxy（本次用 SNR）上呈現**可分離的分佈/均值差異**（P3）。
- **H2（動力學 / quantizer 被繞過的症狀）**：hot code 只是 collapse 症狀；split 會產生形式上變多的 codes，但很快又**只剩單一 child 接手**或整體回到集中（P1/P2 不改善）。

---

## 2) 實驗設定與 Gate

- Phase 3-2 gate：先確保 Phase 3-2 至少通過 P1（避免 split 結果不可歸因）。本次 gate **PASS**（詳見 `PROGRESS.md`）。
- 訓練骨架：沿用 Phase 3-2 best skeleton（RVQ L=4, K=2048, EMA update），eval steps 固定在 `0/200/400/600/800/1000`。
- Split 機制：只驗證 **方案 A（Codebook Split）**；方案 B（Hierarchical Branch）未在本階段實作。
- 每個 run 都輸出 `split_history.json`（包含 split step / parent / children / init / std / interval），並在 eval 時輸出 split group 使用統計（child_active_count、group entropy、top-child usage）。

## 3) 結果總覽（對照 ACCEPTANCE）

**判定規則（ACCEPTANCE.md）**
- P0：不 crash + artifacts 完整（含 `split_history.json`）
- P1：每個 split group 在 split 後第一次 eval，`child_active_count >= 2`
- P2：final(step1000) `layer0_top10_mass <= 0.95`、`layer0_used_codes >= 20`、`feature_mse <= 0.1`
- P3：proxy 分離（例：children 均值差異效果量 `abs(d) >= 0.5`）

| Exp | 設計 | P0 | P1 | P2 | P3 | 核心觀察 |
|---|---|---:|---:|---:|---:|---|
| 7a | one-shot split（k=3 @ step0，noise init） | PASS | PASS（step200） | PASS | N/A | 後期 child collapse（step800/1000 child_active=1） |
| 7b | periodic split（interval=200, hot_k=1, k=3） | PASS | PASS（每次 split 後第一次 eval） | PASS | 作為 base | 整體最穩；仍有 transient collapse（step800）但 step1000 恢復 |
| 7c | 7b + embedding repulsion（w=1e-3, sigma=1） | PASS | PASS | PASS | N/A | step1000 child_active_min=2（恢復）但仍可能高度不平衡（entropy_norm=0.262） |
| 7d | proxy=SNR（base=7b，val max_batches=50） | PASS（analysis run） | N/A | N/A | **FAIL** | 只有 1/4 groups abs(d)≥0.5；多數 groups separation 接近 0 |

> 結論（以 Phase 3-3 的目標來看）：**P0/P1/P2 通過 → split 對 anti-collapse 明顯有效；但 P3（speech/noise proxy 分離）在 sample-level SNR 上證據偏弱。**

## 4) Exp7a — One-shot Split（k=3, init=noise）

**Run dir**：`exp_0128/phase3-3/run_exp7a_oneshot_k3_noise_std1e-3_steps1000_L4_K2048_ema_th2_beta1p0_inter0p5_seed42_20260205_034115`

- Split event：`step0 parent=617 -> children=[0,1] (k=3, std=1e-3)`
- Global（layer0）：
  - step200：top10_mass=0.1407、used_codes=1091、feature_mse=0.0387
  - step1000：top10_mass=0.1698、used_codes=1082、feature_mse=0.0339
- Group 行為（關鍵）：
  - step200：`child_active_count=2`，但 top-child 已偏向單一 child（top_frac≈0.83）
  - step800/1000：**child_active_count 退化為 1（collapse）**

**分論**
- one-shot split 可以在早期快速把 hot parent 的 Voronoi cell 切細（P1 PASS @ step200），但**缺乏持續性的反 collapse 動力**：訓練後期容易再收斂回單一 child。
- 若要用 one-shot 做「speech/noise 分離」推論，會被「後期 child collapse」干擾；更適合作為最小可行性（P0/P1）驗證。

## 5) Exp7b — Periodic Split（interval=200）

**Run dir**：`exp_0128/phase3-3/run_exp7b_periodic_int200_hot1_k3_noise_std1e-3_steps1000_L4_K2048_ema_th2_beta1p0_inter0p5_seed42_20260205_035256`

- Split events（見 `split_history.json`）：
  - step200：parent=220 → children=[8,12]
  - step400：parent=523 → children=[7,13]
  - step600：parent=875 → children=[4,11]
  - step800：parent=498 → children=[1,10]
- Final（step1000）：layer0_entropy=8.9774、top10_mass=0.1602、used_codes=1104、feature_mse=0.0338（P2 PASS）
- Group 行為：
  - step800：較早 split 的 groups 出現 **transient collapse**（child_active_min=1）
  - step1000：所有 groups **恢復為 child_active_count=2**（child_active_min=2，entropy_min=0.4714）

**分論**
- periodic split 有效抑制「hot code 再集中」的趨勢：即使中途 transient collapse，後續 split/訓練仍能讓分流再次被用起來（P1、P2 皆 PASS）。
- 在 Phase 3-3 的目標下，Exp7b 是目前最值得當作後續主線 ablation 的版本（anti-collapse / capacity 增補）。

## 6) Exp7c — Periodic Split + Embedding Repulsion（僅加一種正則）

**Run dir**：`exp_0128/phase3-3/run_exp7c_periodic_int200_hot1_k3_repulse_w1e-3_sigma1_steps1000_L4_K2048_ema_th2_beta1p0_inter0p5_seed42_20260205_040845`

- 目的：針對 split group 的 children embeddings 做 repulsion，降低「split 後只剩單一 child 接手」的 collapse。
- Final（step1000）：layer0_entropy=8.9116、top10_mass=0.1860、used_codes=1104、feature_mse=0.0339（P2 PASS）
- Group 行為（step1000）：
  - `child_active_min=2`（全 groups 最終都有 2 個 child 被使用）
  - 但最小 entropy 很低：`entropy_min=0.1818`，其中一組幾乎單一 child 壟斷（top_child_frac≈0.956）

**分論**
- repulsion **沒有消除 transient collapse（step800 仍 child_active_min=1）**，且對「使用分佈平衡」的幫助不穩定：最後仍可能高度偏向單一 child。
- 若要把「child 之間 usage 更平衡」當成目標，下一步更像是需要 **group-entropy/KL 類的 usage-balance loss** 或 **hierarchical gating**，而不是只靠 embedding repulsion。

## 7) Exp7d — Proxy Separation（proxy=SNR）

**Run dir**：`exp_0128/phase3-3/run_exp7d_proxy_snr_from_run_exp7b_periodic_int200_hot1_k3_noise_std1e-3_steps1000_L4_K2048_ema_th2_beta1p0_inter0p5_seed42_20260205_035256_20260205_050353`  
**Base**：Exp7b 的 `final_model.pt` + `split_history.json`

### 方法
- 取 val set（max_batches=50），對每個 sample 計算 **sample-level SNR(dB)**：
  - `SNR = 10*log10( mean(clean^2) / mean((noisy-clean)^2) )`
- 對每個 layer0 token（frame-level code）把該 sample 的 SNR 當作 proxy，累積到每個 code（parent/children）。
- 對每個 split group 比較兩個 children 的 proxy 分佈（本次用 streaming 統計得到 mean/std，並計算 **Cohen’s d**）。

### 結果（P3）
- Summary：abs_d_median=0.128、pass_groups=1/4（門檻 abs(d)≥0.5）→ **P3 FAIL**
- 四個 split groups（children 的 proxy 分離）：
  - parent=220（8 vs 12）：abs(d)=0.083（幾乎無分離）
  - parent=523（7 vs 13）：abs(d)=0.174（弱）
  - parent=875（4 vs 11）：abs(d)=0.557（中等分離；唯一達標）
  - parent=498（1 vs 10）：abs(d)=0.024（幾乎無分離）

### 限制（為什麼 P3 證據偏弱）
- SNR 是 **sample-level**，把同一個值灌到該 sample 的所有 frames，可能會把「局部噪音」訊號平均掉。
- 部分 child token 數偏少（例如 n=40），效果量估計容易不穩。
- 本環境 CUDA init 偶發 error 304，analysis 自動 fallback 到 CPU（不影響統計正確性，但限制可跑更大規模）。

## 8) Gate 決策（Go/No-Go）與下一步

### 是否符合 Phase 3-3 驗收？
- **P0/P1/P2：通過** → Phase 3-3 的 split 機制確實能改善 collapse 並產生實質分流（值得保留/擴展）。
- **P3：未通過（目前證據弱）** → 不能用 sample-level SNR 直接支持「speech/noise 被穩定拆開」這個主張。

### 建議
- **若目標是 anti-collapse（token 多樣性/可用性）**：建議 **GO**，以 Exp7b 作為後續主線（或基準）配置。
- **若目標是 speech/noise 分離（需要 P3 證據）**：目前 **NO-GO（需升級方法）**，建議優先做：
  1) **per-frame proxy**：用 hop=320 對齊後計算 `||noisy-clean||` 的 frame-wise proxy，再對 children 做分佈比較（更貼近「局部噪音」）。  
  2) **Init-B（local kmeans）**：用被指派到 parent 的 encoder outputs 做局部 kmeans 初始化 children embeddings。  
  3) **usage-balance 正則**：對 group token 的 children usage 做 entropy/KL（比 repulsion 更直接）。  
  4) 若仍不穩，再考慮 **方案 B（Hierarchical Branch）** 做顯式 gate。
