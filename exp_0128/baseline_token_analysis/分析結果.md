# Baseline Token Distribution Analysis Results

**實驗**: exp_0112_intermediate / exp_k_v6
**訓練方法**: LoRA Distillation（**Codebook 凍結**，使用 Teacher 的原始 VQ）
**Checkpoint**: epoch 300
**日期**: 2026-01-27
**分析日期**: 2026-02-05

---

## ⚠️ 重要說明：Baseline 使用凍結的 Codebook

**Baseline (exp_k_v6) 的訓練方式**:
- ✅ Student Encoder: LoRA 微調（rank=256, alpha=512）
- ❌ **VQ Codebook: 完全凍結**（使用 Teacher 的預訓練 codebook）
- ✅ Decoder: 凍結（使用 Teacher 的預訓練 decoder）

**Phase 3-2/3-3 的改變**:
- ✅ Student Encoder: RVQ（殘差向量量化）
- ✅ **VQ Codebook: 重新訓練**（使用 EMA 更新 + hot-code split）
- ❌ Decoder: 仍然凍結（使用 Teacher 的預訓練 decoder）

**這個差異的意義**:
1. **Baseline 的 collapse 分析有限制性**: 因為 codebook 是凍結的，Student 只能選擇 Teacher 已有的 codes，無法學習新的 codebook
2. **Token accuracy 極低的原因**: Student encoder 輸出的 features 可能與 Teacher 不同，但被迫使用相同的凍結 codebook 量化，導致 token mismatch
3. **Phase 3-2/3-3 的優勢**: 重新訓練 codebook 讓 Student 可以學習適合自己 encoder 的量化表示

---

## Executive Summary

本分析檢驗了 **exp_k_v6 (epoch 300)** baseline checkpoint 的 token 分布特性，發現：

1. ✅ **Teacher 模型表現良好**: Val entropy 9.02 bits, top-10 mass 12.25%
2. ❌ **Student 嚴重 collapse**: Train entropy 8.28 bits, top-10 mass **32.27%**
3. ⚠️ **Token accuracy 極低**: Val accuracy 僅 **0.89%**（Student 無法預測 Teacher 的 token）
4. 🔍 **Train collapse 比 Val 更嚴重**: 表明過擬合，Student 在訓練集上學到了捷徑
5. 🔑 **關鍵限制**: Codebook 是凍結的，Student 被迫使用 Teacher 的 codebook

## 5. Key Findings & Implications

### 5.1 Token Accuracy 的真相

從 checkpoint metrics:
```
train_masked_acc: 0.0346 (3.46%)
val_masked_acc: 0.0089 (0.89%)
```

**這意味著**:
- Student 預測的 token 只有 **0.89%** 和 Teacher 一致
- Student 幾乎完全失敗於 token-level distillation
- 但 feature loss 還可以（Val: 0.2475），所以 encoder features 還有學到東西

### 5.2 Collapse 與音質的關係

**重要觀察**: Teacher 有明顯 collapse (Train top-10 = 25.19%)，但音質很好

**可能的解釋**:
1. **語義結構的學習**: Teacher 的 collapse 反映了音頻的實際結構
2. **Decoder 適應**: Decoder 學會了從這些高頻 token 重建出高品質音頻
3. **Good Collapse vs Bad Collapse**:
   - Good (Teacher): 學習到有意義的語義表示
   - Bad (Student): 過擬合，無法泛化

### 5.3 Student 的真正問題

**根本原因：Codebook 凍結**
1. ❌ **Encoder-Codebook 不匹配**: Student encoder 的 features 與 Teacher 不同，但被迫使用相同的凍結 codebook
2. ❌ **Token 預測失敗**: 只有 0.89% accuracy，因為 Student 的 features 無法正確映射到 Teacher 的 codes
3. ❌ **過擬合**: Train/Val 分布不一致（T278 vs T1834）
4. ❌ **泛化能力差**: 無法學習新的 codes 來表示自己的 features

**表面症狀**:
- Collapse: Top-10 mass 32.27%
- 低 accuracy: Val 0.89%
- Train/Val gap: +11.91% top-10 mass

### 5.4 Phase 3-2/3-3 RVQ 的意義

**從 LoRA Distillation (凍結 codebook) → RVQ Retraining (重新訓練 codebook)**

**Baseline 的限制**:
- ❌ Codebook 凍結 → Student 無法學習適合自己的 codes
- ❌ Token accuracy 0.89% → Encoder-Codebook 不匹配
- ❌ 無法衡量 reconstruction quality → 使用 Teacher 的 decoder

**Phase 3-2/3-3 RVQ 的改進**:
- ✅ **重新訓練 codebook**: 使用 EMA 更新，讓 codebook 適應 Student encoder
- ✅ **Hot-Code Split**: 主動分裂高頻 codes，檢驗「訊號/噪音混合」假設
- ✅ **RVQ 分層表示**: 4 層 RVQ (2048 codes/layer)，提供更豐富的表示能力
- ✅ **可測量 reconstruction**: 因為 codebook 與 encoder 一起訓練

**目標**:
1. ✅ 降低 Train/Val collapse gap（改善泛化）
2. ✅ 提升 codebook 與 encoder 的匹配度
3. ✅ 透過 hot-code split 檢驗是否能分離訊號/噪音
4. ⚠️ 不一定追求極高的 token diversity（Teacher 也只有 entropy 9.02）

## 6. Recommendations

### 6.1 Evaluation Metrics

**應該追蹤**:
1. ✅ **Reconstruction Loss** (L1/L2): 音頻重建品質
2. ✅ **Token Accuracy**: Student 預測 Teacher token 的準確率
3. ⚠️ **Entropy/Top-K Mass**: 作為輔助指標，不是主要目標

**不應該作為主要指標**:
- ❌ Entropy 接近 12.0（Teacher 也只有 9.02）
- ❌ Top-10 mass 接近 0.24%（Teacher 也有 12.25%）

### 6.2 Future Work

1. **分析 code-SNR 關係**
   - 使用 Phase 3-3 checkpoints
   - 檢查訊號/噪音是否被分到不同 codes

2. **測量 Reconstruction Quality**
   - 計算 reconstruction loss
   - 使用 PESQ/STOI 等音質指標

3. **理解 Collapse 的語義**
   - 為什麼 T278/T453 等高頻 tokens？
   - 它們代表什麼音頻特徵？

## 8. Conclusion

**Baseline (exp_k_v6 epoch 300) 的狀態**:

| Aspect | Status | Evidence |
|--------|--------|----------|
| Token Diversity | ❌ Poor | Entropy 8.28-9.02 (target: 12.0) |
| Token Collapse | ❌ Severe | Top-10 mass 23-32% (target: 0.24%) |
| Token Accuracy | ❌ Failed | Val: 0.89%, Train: 3.46% |
| Overfitting | ❌ Yes | Train collapse > Val collapse |
| Feature Learning | ⚠️ Partial | Feature MSE ~0.25 (not terrible) |

**Phase 3-2/3-3 RVQ Retraining 的正當性**:
- ✅ Collapse 確實存在（32.27% top-10 mass）
- ✅ Overfitting 確實存在（Train > Val）
- ✅ Token accuracy 極低（0.89%）→ 凍結 codebook 不適合 Student
- ⚠️ 但 Teacher 也有 collapse（25.19%）且音質好
- 🔑 **關鍵改變**: Phase 3-2/3-3 重新訓練 codebook，讓 Student 學習適合自己的量化表示

**Baseline 的核心問題**:
1. **Codebook 凍結**: Student 被迫使用 Teacher 的 codebook，導致 token mismatch
2. **Encoder-Codebook 不匹配**: Student encoder 的 features 與 Teacher 不同，但量化器相同
3. **無法學習新表示**: 即使 Student 發現了更好的特徵，也無法創建新的 codes

**Phase 3-2/3-3 的優勢**:
1. **Codebook 重新訓練**: 使用 RVQ + EMA 更新，codebook 可以適應 Student 的 features
2. **Hot-Code Split**: 主動分裂高頻 codes，增加表示多樣性
3. **潛在的音質提升**: Codebook 與 encoder 匹配，reconstruction 可能更準確

**最重要的發現**:
> Baseline 的 token collapse 和低 accuracy 主要是因為 **codebook 凍結**，而非 Student encoder 本身的問題。Phase 3-2/3-3 透過重新訓練 codebook，讓 Student 可以學習適合自己的量化表示，這是根本性的改進。Teacher 的 collapse 不影響音質，因為它的 encoder-codebook-decoder 是一起訓練的整體系統。
