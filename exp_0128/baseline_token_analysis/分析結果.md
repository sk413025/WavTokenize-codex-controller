## ⚠️ 重要說明：Baseline 使用凍結的 Codebook（與 Phase 3-2/3-3 不同）

**Baseline (exp_k_v6) 的訓練方式**:
- ✅ Student Encoder: LoRA 微調（rank=256, alpha=512）
- ❌ **VQ Codebook: 完全凍結**（沿用 Teacher 預訓練 codebook）
- ✅ Decoder: 凍結（沿用 Teacher 預訓練 decoder）

**Phase 3-2/3-3 的重點差異（你目前在做的方向）**:
- ✅ Student Encoder: RVQ（殘差向量量化）
- ✅ VQ Codebook: 可更新（EMA / hot-code split 等）
- ✅ Decoder: 仍凍結（沿用 Teacher 預訓練 decoder）

**這個差異的意義**（就 token 分布分析而言）:
1. Baseline 的 token 分布反映的是「**固定 codebook 下，Student/Teacher 各自落到哪些 codes**」，而不是「可學習 codebook 的最佳分配」。
2. Token accuracy 很低通常代表：Student encoder features 與 Teacher features 不一致時，在同一個凍結 codebook 上量化會 mismatch（不必然等於音質差，但代表 token-level 對齊差）。
